# UI 设计文档

## 一、设计语言

| 要素 | 方案 |
|------|------|
| **主题** | 暗色主题（深棕+米色羊皮纸纹理），仿古代卷轴 |
| **字体** | 标题：思源宋体 / 正文：思源黑体 / 数字：等宽字体 |
| **配色** | 主色：#8B6914(金) / 背景：#1A1206(深棕) / 文字：#E8D5B7(羊皮白) / 强调：#C62828(朱砂) |
| **间距** | 8px 基础网格单位，组件间距 16px |
| **圆角** | 4px (微妙)，与古代硬直线条相衬 |
| **图标** | 自制簡化像素风格图标 |

### 5.1 LeftPanel — 政务菜单（Demo 已实现）

```

  - 手风琴式折叠，默认全部收起

    谍报（招募密探 + 训练女间谍 / 特工列表 / 派遣 / 反间 / 俘虏）
     计谋（三层折叠：L1 美人计·离间·假情报·空城 ✅ / L2 釜底抽薪·调虎离山·暗渡陈仓等 11 计 / L3 国策态势以逸待劳·远交近攻等 8 策·单选开关）

### 5.2 MapCanvas — 地图渲染

```
Konva Canvas 分层渲染：

Layer 1 — 地形底图
  ├─ 中国地形轮廓(十三州)
  ├─ 河流/山脉/湖泊
  └─ 道路连线(城市间行军路径)

Layer 2 — 城市标记
  ├─ 105 个城市圆形标记
  ├─ 颜色 = 所属势力
  ├─ 大小 = 城市规模(人口)
  └─ 右侧附带标签(城市名+兵力数)

Layer 3 — 部队标记
  ├─ 行军中的部队(三角箭头方向)
  ├─ 驻守部队(旗帜图标)
  └─ 鼠标悬停 → 显示 Tooltip(统帅/兵力/目的地)

Layer 4 — UI 叠加
  ├─ 势力边界(粗略渲染)
  ├─ 选中城市高亮边框
  └─ 事件定位闪烁指示

交互：
  - 点击城市 → RightPanel 切换到 CityDetail
  - 右击城市 → 快速菜单(出征/运输/查看武将)
  - 缩放：滚轮 0.5× ~ 2.0×
  - 拖拽平移地图
```

### 6.1 战斗Canvas

```
Layer 1 — 网格
  ├─ 六角格背景(平/林/山/水/墙 颜色区分)
  └─ Z-order：高地 > 平地 > 水面

Layer 2 — 部队
  ├─ 每支部队一个小旗杆 + 图标
  ├─ 旗帜颜色 = 势力
  ├─ 选中部队 → 绿色六角格高亮
  └─ 移动范围 → 淡黄色六角格高亮

Layer 3 — 特效
  ├─ 火(闪烁红黄色)
  ├─ 水(波纹)
  ├─ 陷阱(灰色隐藏)
  └─ 落石(褐色)

交互：
  - 点击己方部队 → 选中 + 移动范围显示
  - 点击移动高亮格 → 移动
  - 点击可攻击的敌方 → 攻击确认
  - 右击部队 → 部队信息 mini 面板
```

### 6.2 BattleCommandBar

```
底部操作栏(高 60px)：
  [移动]     — 点击后进入移动模式, BFS高亮显示可移动格
  [攻击]     — 点击后进入攻击模式, 显示可攻击敌方(红色)
  [火计]    — Demo 已实现：选我军 → 火计 → 点敌军（气力≥30）
  [计策 ▼]  — 全量设计：水计/落石/伏兵/…（后置）
              灰色=不可用(气力不足/条件不满足)
  [单挑]     — 向相邻敌将发起单挑(灰色=不可用)
  [阵型 ▼]  — 下拉选择可用阵型(当前+切换消耗)
  [待机]     — 结束本队行动
  [全军撤退] — 所有部队撤退(防高/统高时成功率)
```

### 9.1 新游戏 → 主界面

```

2. 每次操作 → API调用 → Zustand store更新 → 地图重绘
3. 操作完成 → 点击"结束回合"
4. Server 处理 AI 回合 → 通过 WebSocket 逐势力推送变化

- 版权：中国古代文物图案已进入公共领域

**前端组合**（react-konva，复用现有架构零新依赖）：

```typescript
// shared/pcg/avatarRenderer.ts（Phase 5 实装）
function renderAvatarBase(group: Konva.Group, gene: AvatarGene) {
  // Layer 0: 宣纸背景（PCG 噪声 或 单张底纹）
  group.add(new Konva.Rect({ fill: 'rgba(255, 246, 220, 1)' /* 宣纸黄 */ }));
  // Layer A: 拓片切片
  group.add(new Konva.Image({
    src: `/assets/rubbings/${gene.baseRubbing}_base.png`,
    globalCompositeOperation: 'multiply',  // 混合模式：拓片墨迹融入宣纸
    filters: [Konva.Filters.Hue, Konva.Filters.Contrast],
    hue: gene.royalSeal ? 10 : 0,
    contrast: 0.2,
    opacity: 0.85,
  }));
}
```

**版权护城河**：公有领域 + 程序化混合差异化，即便两人用同一张拓片切片，经 hue/contrast/混合模式处理后视觉效果不同。

**种子算法**：

```typescript
// shared/pcg/avatarGene.ts（Phase 5 实装）
function getAvatarGene(officerId: string): AvatarGene {
  const seed = hashString(officerId);  // 固定哈希，不随机
  return {
    scheme: 'procedural',
    faceType: Math.floor(seed * 100) % 5,
    hairType: Math.floor(seed * 1000) % 10,
    beardType: Math.floor(seed * 10000) % 10,
    eyeType: Math.floor(seed * 100000) % 10,
    // ... A/B 字段
  };
}
```

**重点人物手工指定**（不靠哈希，落库 `officers.json` `avatarGene` 字段）：

### 12.3 文教、声教、学派与技艺

- UI 正式用语为：`文教`（教育与育才）、`声教`（城市文化发展值 0~1000）、
  `学派`（儒/道/佛/墨/法/纵横/医，各 0~100）、`技艺与制度`（势力级研发，面板简称“技艺”）。
- 城市入口为 `内政 → 文教`：兴办教育、任命学官、建设文化设施、查看学派、设置城市文化政策。
- 势力入口为 `朝廷 → 国策`：选择技艺研发、查看五条路线、设置势力文化政策、查看全国文教概况。
- 任教学官是持续职务，直到撤换、调动、出征或城市陷落；不要求玩家每季重复点击。

- 学派是城市性格，通过设施、学官、政策、事件、迁徙、战争与邻城传播间接改变，不提供“花钱直接 +10”按钮。
- 学派值合计不封顶，因此展示为 `55/100`，不得使用百分比符号或饼图暗示总和为 100%。
- 普通城市教育建筑称“学舍”；“太学”是首都或特定高级城市设施，不与普通 Academy 重名。

### 12.4 状态变更命令的统一终审询问窗

所有会改变游戏状态或推进时间的命令，必须经过：

```text
选择命令 → 侧边抽屉配置参数 → 点击“下达命令” → 居中终审询问窗
                                               ├─ 确认：提交服务端
                                               └─ 取消：返回配置且保留参数
```

终审窗必须列出：命令与类别、执行者、目标、立即/持续消耗、耗时与预计完成季、占用的武将/设施/军队、
主要收益、成功率或情报估算区间、风险与不可逆后果。每条命令必须回答“谁执行、对谁执行、消耗什么、
需要多久、可能发生什么”五个问题。

- 浏览城市/武将、切换标签、缩放拖动、切换图层等只读行为不询问。
- 取消终审保留已填参数；关闭整个命令抽屉才清空草稿。
- 确认前不得调用状态变更 API；提交后按钮锁定并显示“传令中”，防止重复提交。
- 服务端拒绝时保留窗口与参数，显示可操作的失败原因。
- 不提供“以后不再询问”。`Esc`只取消；重大或不可逆命令不得用 `Enter`直接确认。
- 普通确认使用朱砂色；宣战、解盟、处决、废立、称帝、覆盖/载入存档等使用深红危险样式。
- 批量命令只弹一次总确认，但必须逐项列出目标、总消耗与冲突。
- 突发事件的“接受/拒绝”和命令终审视觉同源、语义分离；双轨单挑的“应战/避战”复用事件抉择框。

### 12.5 加强版“结束本季”
