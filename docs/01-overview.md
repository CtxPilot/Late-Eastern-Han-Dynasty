# 项目概述

## 一、项目目标

用 Web 端技术（React + TypeScript + Node.js）构建的现代回合制三国策略游戏。在保留经典三国策略玩法核心精神的基础上，扩展数据规模与系统深度。

## 二、设计理念

| 维度 | 策略 |
|------|------|
| **经典玩法** | 保留核心玩法——回合制策略、武将养成、战斗战棋、隐藏属性系统 |
| **数据扩容** | 1000+武将 / 105城 / 18阵型 / 21兵种 / 165宝物 / 90+女性角色 |
| **系统深化** | 主副将与参谋编成(独立槽位·智≥85·幕僚不带兵) / 爵位编成加成(关内侯→帝·7级) / 部曲系统(12史载武将) / 部队品质(经验Lv1-7+组织度+士气深化) / 军屯田·家属质任制 / 民屯田 / 主副武器双槽 / 兵种适性 / 宝物专属共鸣+套装 / 技能5级制 / 80专属技 / 三轨官职(文24+武44+爵7) / 20级功绩 / 体力系统 / 单挑大会(S19) / **委任军团(地域划区·自动内政出征·官职爵位门槛)** |
| **历史还原** | 婚姻制度遵循东汉史实、女性仅祝融可出战、子女全部取材于史书/演义/传说 |
| **零自创** | 杜绝当代自创武将/女性/子女/剧情 |
| **美术基调** | **金石水墨·拓片简册·印信官职**三件套，公有领域唯一（汉代画像砖/帛画/石刻拓片/竹简/官印/印绶）；独立开发+免版权死命令下的版权护城河；详见 `00-dev-constitution.md` §十一、`07-ui-design.md` §11.6 |

玩法按 **大系统** 划分与推进（非想到哪做到哪）。完整清单、成熟度、扩充顺序见：

**`docs/12-system-map.md`**

摘要（22 大系统）：… · **美女资源** · … · **计谋（战略）** · **历史女角与婚姻** · … · **单挑大会** · … · **美术基调·金石水墨免版权(S22)**  
- **美女 S09** = 势力库存；寻访+1/可寻−1；抢夺多拿降民忠。

## 三、核心差异化

| 对比维度 | 参考设计 | 本作 |
|---------|:--------:|:---:|
| 武将数 | ~500 | **1000+** |
| 属性维度 | 武/智/政/魅（4项） | **统/武/智/政/魅（5项）** |
| 城市数 | ~46 | **105（十三州全部郡国）** |
| 阵型 | 12种 | **18种（孙膑十阵+八阵）** |
| 兵种 | 无独立系统 | **9基础 + 12特殊** |
| 技能 | 6种无等级 | **69通用×5级 + 80专属** |
| 宝物 | ~20件 | **165件（主副双槽+消耗品+先秦珍藏）** |
| 女性 | 个位数 | **90+（婚姻系统+六维影响力+美人赏赐）** |
| 子女 | 无 | **史实出生事件表+母教加成** |
| 单挑 | 3指令 | **7指令+三向克制+全自动结算+宿命对决+武魁大会** |
| 官职 | 单轨 | **三轨分立(文24+武44+爵7)** |
| 经济 | 简单工资 | **人物+军队+城市+农村+运输全消耗链** |
| 体力 | 统一100 | **差异化体力+文武消耗不对称+多恢复途径** |

- **前后端分离架构**
- 浏览器端运行（桌面/平板），纯本地后端
- 单机游戏，无多人对战需求
