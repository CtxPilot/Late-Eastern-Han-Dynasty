## 一、回合系统

### 1.1 时间推进

```
1 年 = 12 个月 = 4 季
1 季 = 3 个月

Game Loop:
  Spring (1~3月) → Summer (4~6月) → Autumn (7~9月) → Winter (10~12月)
  每季 = 1 个操作回合
  每 4 个操作回合 = 1 年
```

### 1.2 季节影响

| 季节 | 正面效果 | 负面效果 |
|:----:|---------|---------|
| **春** | 农业收入+20%，人口自然增长 | 洪水概率+10% |
| **夏** | 商业收入+10% | 瘟疫概率+5%，行军粮耗+20% |
| **秋** | 农业收入+20%(收获) | 台风(南方)+5% |
| **冬** | 征兵效果+10% | 行军移动-1格，北方暴雪 |

### 1.3 回合阶段

```
开始阶段
  ├─ 计算资源收入(金/粮/人口)
  ├─ 自然灾害判定
  ├─ 寿命运作（自然死亡）
  └─ 回合事件触发器

玩家行动阶段
  ├─ 内政(开发/征兵/训练/搜索/赏赐/任命)
  ├─ 军事(出征/运输/撤退)
  ├─ 外交(同盟/联姻/进贡/请求援军/劝降)
  └─ 人事(登用在野/录用俘虏)

结果阶段
  ├─ 衰老(年龄+1季)
  ├─ 疾病判定
  ├─ 战斗结算
  └─ 事件状态清理
```

### 2.8 民屯田

> 民屯田不同于农业开发（`stats.farm`）——农业开发花金升级耕地质量，民屯田是分配多余人口到闲置土地进行耕作。
> 两者平行运作，产粮叠加。

**机制**：
```
每城可分配人口进行民屯:
  maxHouseholds = floor((totalPopulation - reservedLabor) × 0.3)
  实际分配户数 = min(maxHouseholds, 城主可调配额)

产粮公式:
  foodProduced = floor(assignedHouseholds × 0.8 × seasonMul × regionFactor)
  seasonMul: 春×1.0, 夏×1.2, 秋×1.5, 冬×0.6
  regionFactor: 见下方地域系数表

无金消耗（与农业开发不同——农业开发需金100/次）
但占用人口（这些人口不参与商业/征兵/其他劳动）
```

**地域系数**：
| 地域 | 系数 | 说明 |
|:----:|:----:|------|
| 中原（司/兖/豫/青/徐） | 1.2 | 平原沃土 |
| 河北（冀/幽/并） | 1.0 | 华北平原 |
| 关陇（雍/凉） | 0.8 | 干旱偏冷 |
| 荆襄 | 1.1 | 江汉平原 |
| 江东（扬） | 1.0 | 江南水乡初垦 |
| 巴蜀（益） | 0.9 | 山地多平地少 |
| 交广（交） | 1.0 | 湿热一年三熟可能 |
| 南中 | 0.6 | 偏远未垦 |

**切换**（行政指令，每季限1次）：
```
分配户数可调: 0 ~ maxHouseholds（滑块或直接输入）
开/关: 投入人口即开，归零即关
无冷却
```

**与农业开发的比较**：
```
农业开发(stats.farm):   花金 → 升farm值 → 产粮 = farm×3.2×laborFactor×season
                         优点: 不占人口，效率高
                         缺点: 花金，有上限(999)，需持续投入

民屯田:                 分人口 → 直接产粮 = 户数×0.8×season×region
                         优点: 不花金，无上限（受人口约束）
                          缺点: 占人口（减少劳动力和征兵池）
```

### 2.9 当前 Demo 实装与随机性边界

> 本节用于区分上文 v3 完整设计与当前运行时代码，避免把设计稿误报为已实现。

- 当前 `server/src/engine/civil.ts` 仍是即时结算切片：农业、商业、城防、征兵、施米、训练已实现。
- 文化、工艺、交通、卫生的持续任务，以及军屯田、民屯田仍仅有设计，尚无运行时随机路径。
- 六条已实现路径均由服务端编排层显式注入单一权威 `xorshift32-v1`；引擎函数没有 `Math.random()` 或随机默认参数。
- 固定保存点执行六条指令会严格消费 6 次随机数；恢复后增益序列、最终城市状态和 `draws` 必须一致，由 `pnpm verify-civil-rng`（12/12）守护。

## 三、人事系统

> **S11**。**仅男将**（`officers`）；历史女角禁止搜索/登用（§30.6，归 S18）。  
> **已落地（Demo）**：`searchTalent` / `recruitOfficer` · `POST /personnel/search|recruit` · PersonnelPanel。

### 3.1 搜索 — **已实现（Demo）**

```
消耗：金 80（己方城）
搜索者：本城智+魅最高己方将，否则任意现役/君主
成功率 = clamp(智/150 + 魅/200, 15%, 85%)

成功结果池：
  有本城/邻接/无坐标在野将时 ~70% 发现该将（迁至搜索城，仍 FREE，可登用）
  否则/其余：小额金 / 粮 / 无才可录
失败：无果仍扣金
```

### 3.2 登用 — **已实现（Demo）**

```
成功率 = 基础40%
  + (魅力 - 目标魅力) * 0.3
  + (1 - |我方相性 - 目标相性| / 150) * 40%
  + 目标义理 * 0.02
  - 目标野心 * 0.03
  clamp [5%, 90%]

登用在野武将：金 200；说客默认同城己方将或君主
成功：joinFaction（妻随从）+ 入城 officers；初始忠诚 ≈ 55+chance/10
失败：仍扣金
录用俘虏：⏳ 需目标忠诚≤30，成功率额外-20%（后置）
```

> **特殊人物不可录用原则**：方士/术士/纯传说人物（如左慈、于吉、管辂等）不录入 `officers.json`，不可通过搜索/登用录用。此类人物的剧情引用通过事件系统（S14）实现。华佗已存在于数据中，属既有事实，后续不再新增同类。**例外**：祝融作为游戏设计明确的可出战女将，正常录入。

## 四、出身与关系网系统

### 4.1 出身标签

每名武将以多维标签定义出身，而非单一分类。标签分5类，每将3~6个标签：

**社会出身（7种）**：宗室/外戚/士族/豪族/寒门/平民/商贾

**出身地域（10区）**：中原/河北/关陇/并幽/青徐/荆襄/江东/巴蜀/南中/交广  
**边地修饰**：并幽/关陇/南中的武将额外可打 `边地` 标签（军功晋升+20%，文化产出-1/季）

**职业背景（11种）**：官吏/武人/儒生/游侠/商贾/医者/方士/叛将/名士/黄巾旧部/蛮族

**政治立场（6种）**：匡扶汉室/天命革新/割据自守/择木而栖/名利之徒/苟全性命

**特殊标签（6种）**：宦官之后/弑主/二主之臣/宗室远支/义兄弟/父子

**出身标签的机械效果**（§4.5 社会阶层联动表）：

| 社会出身 | 教育倍率 | 政治特权 | 副作用 |

### 4.5 出身对游戏的核心影响

> 出身是与五维**平行运作**的第二套体系。袁绍统武平平却能雄踞河北，不是因为他强，是因为他姓袁。
> 以下规则让出身成为与属性同等重要的决策变量。

#### 4.5.1 仕官门槛与天花板

不同出身的武将初始官职和晋升天花板不同：

| 出身 | 初始官职下限 | 仕官天花板 | 登用倾向 |
|:----:|:----------:|:----------:|:--------:|
| **士族** | 县令/校尉 | 丞相/大将军 | 投奔士族势力+30%，投奔寒门-20% |
| **豪强** | 军候/书吏 | 将军/九卿 | 投奔本地势力+20% |
| **宗室** | 县令/将军 | 君主 | 投奔宗室势力+50% |
| **寒门** | 无职/军候 | 将军/郡守（君主特批可破格） | 投奔寒门/平民势力+20% |
| **平民** | 无职 | 校尉（军功可破格至大将军） | 投奔任何势力 |
| **商贾** | 无职 | 郡守（不可任大将军/丞相） | 投奔重视商业的势力+15% |

**君主出身对势力整体影响**：
```
士族君主（袁绍/刘表/孙权）:
  登用士族成功率 +30%
  登用寒门成功率 -20%
  外交初始友好 +15（士族间惺惺相惜）
  称帝合法性 +20%

寒门君主（刘备/曹操[宦官之后被视为寒门以上豪强以下]）:
  登用寒门成功率 +20%
  登用士族成功率 -15%
  寒门武将忠诚 +5
  称帝合法性 -10（需"禅让"或"献帝"加持）

平民君主（吕布/张角）:
  登用士族成功率 -40%
  登用寒门成功率 +10%
  武将整体忠诚 -5（威望不足）
  称帝合法性 -30
```

#### 4.5.2 派系系统

> 每个势力内部按出身自动形成派系，派系之间的力量对比影响内政效率、忠诚和继承权。

**派系形成**：
```
每季检查势力内武将出身分布:
  士族 ≥ 3 人 → 自动形成"士族派"
  寒门 ≥ 3 人 → 自动形成"寒门派"  
  豪强 ≥ 2 人 → 自动形成"豪强派"
  平民 ≥ 3 人 → 自动形成"平民派"

派系领袖 = 该派系中官职最高者（同官职则看知名度）
```

**派系效果**：
```
同派系武将:
  互相同僚关系 +20/季（加速亲密）
  自动支援概率 +15%
  计策/内政协作 +10%

异派系武将:
  互相同僚关系 -5/季（渐行渐远）
  争官时会互相打压（忠诚-2/次）
  严重时 → 派系内讧事件
```

**经典案例——袁绍的派系分裂**：
```
袁绍势力（河北）:
  士族派: 审配·逢纪·郭图·辛评（河北本地士族）
  寒门派: 沮授·田丰（有才但出身低）
  
矛盾:
  士族派排斥寒门派 → 沮授建议常被否决
  官渡之战时: 派系内斗导致许攸投曹
  袁绍死后: 士族派分裂为袁谭派 vs 袁尚派 → 势力瓦解

游戏表现:
  若玩家是袁绍，需要平衡两派——否则自动触发内讧事件
  若玩家是曹操（寒门+豪强+士族混杂），需用"唯才是举"政策压制派系斗争
```

#### 4.5.3 政治资本

出身影响势力能做的政治操作：

**称帝**：
```
需要士族支持率:
  ≥ 60% → 称帝可行，天下反应：部分势力承认
  40~59% → 称帝后声望-20，部分势力宣战（袁术式悲剧）
  < 40% → 称帝后几乎所有势力宣战

士族支持率 = 势力内士族武将占比 + 外交关系中士族势力的友好度修正
```

**联盟发起权**：
```
反XX联盟:
  发起人条件: 士族出身 或 宗室出身 或 声望≥5000
  袁绍发起反董联盟: 自动获得盟主（他是士族领袖）
  曹操发起反XX: 需要外交游说（他是宦官之后）
  刘备发起: 以"汉室宗亲"身份可行（宗室远支仍有号召力）
```

**外交联姻**：
```
士族×士族联姻: 友好度 +30
士族×寒门联姻: 友好度 +10（寒门高攀）
宗室×士族联姻: 友好度 +25（政治联姻最高效）
平民势力联姻: 友好度 +5（无政治价值）
```

#### 4.5.4 继承权

出身影响继承人选的合法性和派系支持：

```
嫡子（正妻所生）:
  继承权重 = 100（基准）
  妻族出身加成: 士族+30, 宗室+25, 豪强+10, 平民/寒门+0

庶子（妾所生）:
  继承权重 = 60

养子/义子:
  继承权重 = 30（除非已明确指定）

君主废长立幼:
  若新立之子的母族出身低于嫡子母族 → 士族派叛乱概率+40%
  若新立之子无派系支持 → 内战概率+60%
```

**袁绍继承案例**：
```
袁绍三子:
  袁谭（长子）: 母族平民 → 继承权重 = 100 + 0 = 100
  袁熙（次子）: 母族平民 → 继承权重 = 60
  袁尚（幼子）: **母族士族（刘氏）** → 继承权重 = 60 + 30 = 90

袁绍偏爱袁尚（母族士族），试图废长立幼：
  → 士族派分裂: 审配支持袁尚（同士族），辛评支持袁谭
  → 内战概率判定: 母族出身差=30 + 派系分裂=40 → 70% 触发内战
  → 结果: 河北分裂，曹操逐个击破
```

#### 4.5.5 教育传承（与§34联动）

```
士族子弟:
  自动获得高水平教育（即使城市教育值低）
  成丁时基础属性 + floor(父亲智力/10)（家学渊源）
  16岁起自动获得一个技能（随机，偏向政治/计略）

寒门子弟:
  教育完全依赖城市教育值
  若城市教育值 ≥ 500 → 有机会突破阶层限制（如诸葛亮）
  成丁时无额外技能，但若有天赋（隐藏属性奇谋≥60）→ 自动领悟一个战术技能

平民子弟:
  成丁时基础属性无加成
  但若父亲武力和统帅均≥80 → 有概率继承战斗天赋（+10武/+5统）
```

#### 4.5.6 袁绍案例——出身如何定义一个人物

```
袁绍（id=? 0-B添加）:
  社会出身: 士族（袁氏四世三公——东汉最顶级门阀）
  地域: 中原（汝南袁氏）
  职业: 官吏+名士
  政治: 割据自守（后期）→ 天命革新（早期）
  特殊: 宗室远支（袁氏与汉代有姻亲）

出身给他带来的:
  ✅ 登用河北士族几乎100%成功（审配/逢纪/郭图/辛评）
  ✅ 反董联盟自动成为盟主（士族领袖身份）
  ✅ 开局势力和地盘最大（继承袁氏政治遗产）
  ✅ 外交友好度初始+20（士族网络）
  
出身也限制了他:
  ❌ 寒门人才（沮授/田丰/张郃）始终无法获得他完全信任
  ❌ 派系内斗消耗了大量精力
  ❌ 官渡之战时士族派系不愿配合寒门将领的战术
  ❌ 死后立即分裂——派系矛盾压不住了

玩家操作袁绍:
  要主动压制派系斗争，提拔寒门人才平衡势力
  历史上袁绍失败不是因为能力差，是因为出身决定了他的行为模式
   ——他从未想过要平衡士族和寒门，因为他本身就是士族的化身
```

#### 4.5.7 出身决定命运——经典案例

**案例一：刘表单骑定荆州（名士声望·和平占领）**

```
历史:
  刘表，字景升，党锢名士"八俊"之一
  董卓乱政后，荆州刺史被杀了，朝廷派刘表去接任
  刘表**一个人一匹马**进了荆州（"单骑入荆州"）
  靠名士声望和当地士族（蒯越/蔡瑁）的支持，不费一兵一卒拿下荆州

游戏机制——「名士声望」:
  刘表 tags: 士族+名士+官吏
  「名士」效果: 名望+30，和平占领概率+40%，当地士族自动归附
  
  当派出名士武将赴任空城/乱城时:
    和平接管概率 = 名士声望/100 + 当地士族友好度/50
    刘表接管荆州: 名望80 + 蒯氏友好70 → (80/100 + 70/50) = 0.8+1.4 = 220% → 自动成功
    非名士武将: 无此加成，需带兵硬打

  其他名士案例:
    诸葛亮: 名士+儒生 → 刘备三顾（"卧龙凤雏"的名士声望）
    司马徽: 名士 → 一句话让刘备去找诸葛亮
    许劭: 名士（月旦评创始人）→ 评价曹操"治世能臣乱世奸雄"→ 曹操声望+20
    孔融: 名士+宗室 → 到哪都被奉为上宾
```

**案例二：刘备创业为什么这么难（寒门+宗室远支的双重困境）**

```
历史:
  刘备，中山靖王之后，但到他这代已经沦落到织席贩履
  他是宗室远支（275年前的王室后代，朝廷不认）
  他是寒门（父亲早逝，与母亲靠手工活维生）
  早期只有关羽张飞跟随——因为平民和寒门不看身份，看义气
   投谁谁灭——公孙瓒/陶谦/吕布/曹操/袁绍/刘表，没人敢长期重用他
   但有一件事暴露了他真正的政治智慧：
     陶谦表荐刘备为"茂才"（东汉最稀缺的仕途入场券）
     刘备没有留给自己用，而是转手**给了袁谭**（袁绍的长子）
     这一手等于向袁氏家族递了投名状——一个寒门宗室远支，用自己最珍贵的政治资本，去巴结天下第一士族
     所以官渡之战时，刘备毫不犹豫去投奔袁绍——不是袁绍本人，是因为那层早铺垫好的关系
   直到诸葛亮加入（士族背书），才开始有士族人才愿意投靠

游戏表现:
  § 4.5.1 规则验证:
    刘备(寒门+宗室远支): 登用士族成功率 -20%（没人看好他）
    早期确实只能用关羽(寒门)/张飞(平民)/赵云(平民)
    投靠的势力(公孙瓒/陶谦)都是因为他的人品而非出身
  
  转折点: 诸葛亮加入后 → 士族背书 → 登用士族成功率从 -20% → +0%
  因为诸葛亮是荆州士族圈的核心人物，他的加入意味着"刘备被士族认可了"

  玩家操作刘备:
    早期核心: 依靠义兄弟(关羽/张飞)的关系加成优势
    中期: 必须拿下诸葛亮（同时获得"名士背书"效果）
    后期: 用汉室宗亲身份争取合法性（称帝/联刘表）
    一生限制: 无法像曹操那样做到"唯才是举"——他以仁义立身，不能过度打压士族
```

**案例三：荀彧与颍川谋士团（地域人才网络）**

```
历史:
  荀彧投奔曹操后，做的第一件事不是献策，而是**写信**
  他把颍川同乡的士族好友一个一个叫来:
    荀攸(侄子) → 郭嘉(同乡) → 陈群(同乡) → 钟繇(同乡) → 戏志才(同乡)
  这不仅是个人推荐——这是颍川士族网络的"集体站队"
  曹操由此获得了当时最顶级的人才库

游戏机制——「地域人才网络」:
  每个地域有一个隐性的"人才网络":
    颍川(中原):荀彧→荀攸→郭嘉→陈群→钟繇→戏志才
    河北:审配→逢纪→辛评→郭图→田丰(半边缘)
    江东:张昭→诸葛瑾→顾雍→步骘→虞翻
    荆襄:蒯越→蔡瑁→庞统→诸葛亮(半边缘)
    关陇:法正→张松→孟达

  网络效应规则:
    若A(士族+中原)被登用 → B(士族+中原)登用成功率+25%
    若C(士族+中原)已在一势力 → D(士族+中原)投奔该势力倾向+30%
    跨网络: 士族只看地域（河北士族不自动投奔中原势力）

  曹操的"唯才是举"政策:
    可**打破**地域和阶层限制——但代价是士族忠诚-5

### 7.1 并非统一100

```

## 十二、经济与消耗系统

> **Demo 真源**：城池 `city.gold` / `city.food` 为权威；`faction.gold` / `faction.food` 为汇总缓存。  
> `syncFactionResources`（`economy.ts`）在 **开局 / 进贡·结盟 / 占城 / 回合末** 全量同步，避免缓存陈旧。

### 8.1 收入来源

商税(commerce×人口×税率) + 农税(farm×人口×税率+季节) + 关税(边境/港口城) + 朝贡(附属势力)

### 8.2 人物俸禄（每季）

文官(5~150金/季，Lv1→24) + 武官(5~250金/季，Lv1→44) + 爵位(0~400金/季，7级)
三轨可叠加：如关羽(前将军140 + 汉寿亭侯50) = 190金/季

### 8.3 军队消耗（每季）

粮耗 = 兵数 × 单位系数 × 状态倍率 × 季节
兵种系数：轻步×1.0 / 重步×1.3 / 骑兵×2.0~2.5 / 象兵×5.0 / 攻城器械×3.0
状态倍率：驻守×1.0 / 行军×2.0 / 战斗×3.0
装备磨损：耐久 -2~5%/季，<50%攻-10%，<20%攻-20%+士气-5

### 8.4 城市运营 & 农村消耗

城市：城墙维护/官署/粮仓/驻军设施/市场/治安（共7项）
农村：种子粮/水利/农具/道路/牛马喂养/赈灾储备/徭役（共7项）
季节专项：春耕种子、夏防汛、冬取暖

### 8.5 运输损耗

d≤5格损耗5% → 10格12% → 15格25% → >15格40%。雨季冬季+5%。

#### 贸易路线

每城上限2~4条路线。相邻陆路= (commerceA+commerceB)/200×交通/100×技能。同河运×1.3。跨势力×0.7(关税,友好每10恢复1%)。丝绸之路(长安→敦煌→西域)×2.0。交广海路×1.6。设施：驿站(陆路+15%)/码头(水运解锁)/市集(路线上限+1)/通关口(关税+30%)。特产异地溢价30~120%。禁运对敌自动生效。劫掠路线抢当季贸易收入30%(友好-15,对方获宣战理由)。

## 十七、事件系统

> **条件式历史引擎 + 选项 UI 已落地**（Session 106）：
> - `tickEvents` 只扫描当前 Scenario.eventIds，并按 `sourceClass` 过滤启用史料层

### 17.1 事件类型与史源

| 类型 | 触发时机 | 示例 |
|------|---------|------|
| **历史事件** | 条件满足时触发 | 陈留起兵、迁都长安、汴水追击 |
| **随机事件** | 每季概率触发 | 丰收/灾荒/瘟疫/在野人才来访 |
| **婚配事件** | 特定女性+特定时间 | 二乔配孙策/周瑜 |
| **外交事件** | 外交状态变化触发 | 联姻成功/盟约到期/宣战 |
| **战场事件** | 战斗中特定条件触发 | 单挑、落凤坡伏击 |

玩法类别 `category` 与史源 `sourceClass` 分离。史源固定为正史、裴注及早期异闻、文学演义、民间传说、玩法改编五层；事件弹窗必须显示来源。文学/传说默认开启但可在 ScenarioSelect 关闭，不得把“连环计”“三英战吕布”等文学情节标成正史。

### 17.2 条件式事件链

```
重大事件不能由年份单独触发：
  史实线：时间窗 + 关键人物/势力/据点 + 前置全部成立
  变体线：事件成立但由玩家或AI选择不同处置
  被阻止：核心条件不成立，继续等待至窗口结束
  已失效：超过窗口、决策势力灭亡或互斥分支已完成
```

## 二十四、伤兵系统

### 24.1 战场伤亡分流

```
每场战斗中损失兵力的分配：

  阵亡 = 损失 × 30~50%(永久消失)
  伤兵 = 损失 × 50~70%(可恢复)

伤兵跟随部队：
  行军时携带伤兵 → 粮耗+30%(额外负担)
  伤兵不可战斗
  部队全灭 → 所有伤兵转为阵亡
  撤退成功 → 伤兵保留，带回城市
```

### 24.2 伤兵恢复

```
每季恢复率(在城市中)：

  基础 = 10%
  + 医馆设施 +15%/季
  + 训练将医术技能 Lv×3%/季
  + 城市卫生/30 (%)
  + 守将有太平清领道 +10%/季
  + 华佗/青囊经 +20%/季

恢复数 = 伤兵数 × 恢复率(上限80%)

若不治疗：
  伤兵恶化率 = 5%/季
    - 医馆 -3%
    - 医术技能 -2%
  恶化 → 伤兵死亡(阵亡)
```

### 24.3 伤兵容量

```
最大伤兵携带量 = 部队总兵力 × 30%(正常行军)
若伤兵超过容量→每季超出部分50%恶化(死亡)

若在城市中 → 容量不限(有民房、医馆)
```

### 24.4 战后伤兵统计

```
战后结算画面：

  本次战损: 阵亡 1200 + 伤兵 1800 = 总损 3000
  当前部队: 5000兵 + 1800伤兵(待恢复)
  预计恢复: 1800 × 15%/季 = 270/季 → 约7季全愈
  建议: 返回城市治疗(恢复率+医馆15%=30%/季→约4季全愈)
```

## 二十五、部队等级系统

> **本设计与 `05-combat-system.md §5.6.1` 内容一致。实装以 05 §5.6 为真源**，此处保留为历史设计参考。

### 25.1 七级部队

| Lv | 名称 | 所需经验 | 攻 | 防 | 士气上限 | 特性 |
|:--:|------|:--:|:--:|:--:|:--:|------|
| 1 | 新卒 | 0 | — | — | 100 | 默认，从城市招募 |
| 2 | 训练兵 | 100 | +5% | +5% | 105 | — |
| 3 | 老兵 | 300 | +10% | +10% | 110 | — |
| 4 | 劲旅 | 600 | +15% | +15% | 115 | 暴击率+2% |
| 5 | 精锐 | 1000 | +20% | +20% | 120 | 暴击率+4%、不易混乱 |
| 6 | 百战 | 1800 | +25% | +25% | 125 | 士气永不低于50 |

### 25.2 经验获取

```
每场战斗：
  部队参战 +20xp(基础)
  击破敌方一队 +30xp(每队)
  对方主将被斩/俘 +50xp
  对方总兵力远超己方 +0.3倍奖励(以少胜多)
  胜仗存活 ×1.5(区别于撤退)

部队全灭：
  经验归零 → 回到新卒(除Lv7铁军可保留30%经验)

战斗中不可升级，升级发生在战后结算。
```

### 25.3 补员稀释

```
当部队从城市补充新兵时，经验按加权平均稀释：

  新经验 = (旧兵数×旧xp + 新兵数×0) / (旧兵数+新兵数)

例：
  800劲旅(600xp,+15%攻) + 补充200新卒(0xp)
  → 新xp = (800×600 + 200×0)/1000 = 480 → 降为老兵Lv3(300xp→无额外效果变化,Lv维持)
  → 若补充400新卒: 新xp=480000/1200=400→仍是老兵Lv3
  → 若补充1000新卒: 新xp=480000/1800=267→降为训练兵Lv2(200跨过)→Lv保持老兵但加成减弱
```

### 25.4 训练加速

```
在城市中执行"训练"指令，除提升士气外还可提升经验：

  训练1次 = +5xp(满兵, 消耗金200)
  连续训练在同一季中=效果递减(第2次+3xp,第3次+1xp)
  训练武将的统/100 × 5 → 额外经验(统100=+5xp)

鼓励但不替代实战：全部通过训练到Lv7不可能(最多到Lv4劲旅)
```

### 25.5 部队等级与兵质的关系

```
部队等级(Lv1~7) → 战斗经验决定的战斗力(攻/防/士气/暴击)
兵质(Lv0~3)     → 招募+训练决定的装备质量(攻+5%/级,防+5%/级)

两者独立加成，可叠加：
  铁军(Lv7,+30%攻) × 精锐兵质(Lv3,+15%攻) = +49.5%攻
```

### 25.6 部队命名与状态

```
部队ID = 主将名 + 兵种 + 等级称号
例: "关羽·重骑兵·百战" (Lv6, +25%攻防)

部队旗帜上星标显示等级(Lv4+★1, Lv5+★★, Lv6+★★★, Lv7⋆)
```

### 26.7 与原有四层的区别总结

```
通用技能: 需主动使用 ("我选择放火计")
专属技能: 独有招牌 ("只有吕布有无双")
特性:     被动·条件触发 ("兵力<50%→自动触发铁壁")
隐藏属性: 纯数值 ("勇猛=7")
出身标签: 身份分类 ("寒门+青徐")
```

> **暴击/反击/连击是特性、通用技能、专属技能三者在战场上交汇的结果**：特性决定被动阈值与成长曲线，通用技能决定可培养的倾向，专属技能决定独特战术定位。三者共同构成武将的战斗风格差异（如：张飞=反击爆发型，典韦=双反击护卫型，马超=冲锋连击型，关羽=单发高暴击型，赵云=单骑全能型）。

### 26.8 SkillEffect type 扩展：暴击/反击/连击

```typescript

### 27.2 天花板隐藏加成（原理 + 数值）【定稿】

**原理**（面板属性与隐藏加成分离，保留「断层体感」）：

- 隐藏加成 = **仅五位天花板**的「断层特权」，抬公式输入，不抬整张榜单
- 与专属技 / 金色特性 / 宝物 **叠加不替代**

| 维 | 人物 | 面板 | 隐藏加成 | 裸有效值 | 意图 |

```

## 二十八、人口结构系统

> **状态（2026-07-16）**：**D0~D3 已在 Demo 落地**（`shared/demographics.ts` + `engine/turn.ts` + RightPanel）。  
> **D4**（瘟疫/屠城/俘获/伤兵回流）未做。  
> **原则**：人口不是装饰数字，而是征兵、增长、经济、灾疫、美人资源的统一底层。  
> **实现真源代码**：`shared/demographics.ts`（公式常量）；本文为设计说明，改数字须代码+文档双改。

### 28.1 结构定义

城市人口拆为四桶（单位：人）：

| 桶 | 字段 | 含义 | 主要角色 |
|:--:|------|------|----------|
| **成年男** | `adultMale` | 15~59 岁男性（简化） | **征兵池**、农业/劳役主力、**粮耗最高** |
| **成年女** | `adultFemale` | 15~59 岁女性 | **生育**、劳动；**决定 beautyPool** |
| **儿童** | `child` | 0~14 岁 | 未来劳动力；**新生只入此桶**；不征兵 |
| **老人** | `elder` | 60+ | 轻劳动；**自然衰老死亡主因** |
| **美女资源** | `beautyPool` | 城级可搜罗/赏赐的美人资源 | **∝ adultFemale**（强制同步） |

```
总人口 total = adultMale + adultFemale + child + elder
population ≡ total（写路径必须改桶 → withSyncedPopulation）

// 美女资源（Session 33）
beautyPool = floor(adultFemale * BEAUTY_PER_ADULT_FEMALE)
// BEAUTY_PER_ADULT_FEMALE = 1/400  → 约 400 女成 = 1 点资源
// 搜罗美人：耗 1 点池 + 扣 adultFemale 400（互逆），再 withSyncedPopulation，池不会涨回
```

**开局拆分默认比**（`DEFAULT_DEMO_RATIO`，与代码一致）：

| 档 | 男成 | 女成 | 儿童 | 老人 |
|----|:--:|:--:|:--:|:--:|
| **当前 Demo 默认** | **0.30** | **0.27** | **0.29** | **0.14** |
| 边郡/战乱（预留） | 0.30 | 0.26 | 0.28 | 0.16 |
| 富庶腹地（预留） | 0.26 | 0.28 | 0.32 | 0.14 |

`maxPopulation` = 总人口上限；增长时 `headroom` 限制新生。

### 28.2 核心公式（设计真源）

#### A. 人口自然变动（每回合=1 月，在结算阶段）

```
// 可婚配代理（简化，不做精确婚姻图）
couples = min(adultMale, adultFemale)

foodRatio = clamp(库存粮 / 本月结构粮耗, 0.25, 2.0)

// 1) 生育：全部进入【儿童】桶（不直接变成人）
births = floor(couples * 0.012 * foodRatio * moraleMod * seasonBirthMul)
child += births
// 新生儿性别比仅作备注：约 105:100（略偏男）；童桶暂不拆性别

// 2) 儿童本底死亡 → 3) 儿童长成【成人】
toAdult = floor(child * 0.0085)            // 约 10 年童年节奏（游戏可感）
// ★ 长成性别比（史学简化）：约 112 男 : 100 女
//   依据：历代户籍常见「男口多于女口」；史家释为溺女、漏籍、女孩死亡率偏高等
//   成丁男占比 ≈ 112/212 ≈ 52.8%（高于出生 105:100）
adultMale   += floor(toAdult * 112/212)
adultFemale += toAdult - 上述

// 4) 成人本底死亡（女口略高，产育风险简化）
// 5) 成人 → 老人（自然衰老）
// 6) 老人死亡（自然减员主因，冬/缺粮加权）

应用顺序：出生→童亡→长成→丁故→入老→老故→clamp(maxPopulation)
```

**性别比文献取向（设计说明，非逐郡精校）**：
| 阶段 | 比例 | 说明 |
|------|------|------|
| 新生儿 | ≈105♂:100♀ | 生物学常识；古代亦以略偏男计 |
| 儿童→成人 | ≈112♂:100♀ | 参考传统户籍男多女少（约 110~120 区间的中位简化） |

**用户需求对齐**：
- 新增人口必须是 **儿童**，长成需时间  
- 自然衰老导致 **老人死亡 → 总人口下降压力**  
- 成丁男女比 **参考古代户籍偏男**，非 1:1

#### B. 征兵（成年男硬约束）

```
// 可征上限：不能掏空城防与劳作
reservedLabor = ceil(total * 0.08)         // 最低留城男丁
maxConscript  = max(0, adultMale - reservedLabor)

// 单次征兵量
request     = playerInput
actualGain  = min(request, maxConscript,
                  floor(gold/goldPer), floor(food/foodPer))

// 结算：兵 +actualGain；adultMale -actualGain
// 人口 total 同步下降；民心 -f(actualGain/adultMale)
```

**用户需求对齐**：招兵数量由 **成年男性** 决定，不是总人口随便扣。

#### C. 伤亡回流（与伤兵/战后结算挂钩）

```
战死:     adultMale 永久 -dead
伤残回流: 伤兵恢复后 → adultMale 部分回归；重残可转 elder 或永久流失
俘获:     敌方 adultMale 转入己方战俘池（人事/关押系统）
屠城/饥荒: 四桶按脆弱度加权死亡（儿童/老人权重更高）
```

### 28.3 与其它系统挂钩

| 系统 | 挂钩方式 |
|------|----------|
| **经济·粮耗** | 人粮 = **男成0.048** + 女成0.032 + 儿童0.018 + 老人0.024（×季节，冬×1.2）；驻军 **0.055**/人；男成显著高于女成/童/老；缺粮先死老/童 |
| **经济·农产** | 劳力 = 男1.0 + 女0.85 + 老人0.35 + 儿童0.1；产出 ∝ farm × labor / 基准 |
| **经济·商业** | 消费人口 = total；劳动 ∝ 成人；商税 ∝ commerce × 成人 |
| **征兵/兵质** | 上限=男成；募兵制吃男成更少、征兵制吃更多；尚武提高志愿（少扣民心） |
| **民心/施米** | 施米按 total 耗粮；民心低 → birthMod↓、叛乱↑、征兵抗性↑ |
| **卫生/瘟疫** | 卫生低时 elder/child 死亡率↑；男成染疫影响征兵池 |
| **迁民/招抚** | 流民包按比例拆四桶迁入；战乱城输出流民 |
| **美人/赏赐** | `beautyPool ∝ adultFemale`；**搜罗**耗池+女成（Demo 已通）；赏赐/婚配全量仍属 P4 |
| **城等/上限** | `maxPopulation` 约束 total；设施（粮仓/医馆）改耗粮与死亡率 |
| **AI** | AI 征兵保留 `reservedLabor`；饥荒时优先停征兵、施米 |

## 二十九、谍报与反谍报系统

> **代码真源**：`shared/types/spy.ts` · `server/src/engine/spy.ts` · 可见性 `shared/intel.ts`  
> **Demo 切片**：招募/探秘/破坏/刺杀/驻守反间/俘虏处置 + AI 对称；即时结算。

### 29.1 定位

与军事并列：进攻（派密探）+ 防守（反间布防）。特工为**独立池**，不占武将编制。AI 与玩家共用引擎。

### 29.2 人员与编制

| 项 | 规则 |
|----|------|
| 状态 | idle / deployed / counter_duty / recovering / captive / dead |
| 等级 | Lv1~5；经验来自成功任务 |
| 技能 | recon / sabotage / lethal / tradecraft（0~100） |
| 编制上限 | `1 + floor(己方城数/3)`，硬顶 **8** |

### 29.3 招募（审批定稿）

在**一座己方城**招募；**人数与初始等级**由该城 **成年男 + 驻军** 共同决定：

```
laborPool = adultMale + troops
batch = clamp(1, 3, floor(laborPool / 8000))   // 再受编制空位截断
// 每人初始等级：
//   laborPool < 5000        → Lv1
//   5000 ~ 14999            → 70% Lv1 / 30% Lv2
//   ≥ 15000                 → 50% Lv1 / 35% Lv2 / 15% Lv3
费用 = 金 120×batch + 粮 60×batch
// 不扣人口/兵力（选人叙事）；只扣金粮
```

### 29.4 进攻任务

| 类型 | 主技能 | 成功 | 风险 |
|------|--------|------|------|
| 探秘 recon | recon | 目标城 detailed 情报（月数 2+rank） | 低 |
| 破坏 sabotage | sabotage | 粮/金/城防/士气随机扣 | 中 |
| 刺杀 assassinate | lethal | 城内敌将忠诚−25~40 | 高 |
| **枕边风** pillowTalk | tradecraft | 城内敌将忠诚−18~35+rank×2 | 中（仅女间谍） |
| **离间** sowDiscord | tradecraft | 目标势力与第三方势力友好− 或目标城民忠− | 中（仅女间谍） |

**女间谍专属**（§30.5）：枕边风/离间仅限 `agentKind==='female'` 的特工执行。
枕边风/离间在有 detailed 情报时成功率 +10。离间优先找目标城邻接的第三方势力降友好；无第三方则降目标城民忠。

条件：idle、冷却 0、目标非己方、与己方城**官道邻接**；禁止对**同盟**破坏/刺杀。结算**即时**。

### 29.5 反谍报

- 特工 **驻守反间** → 该城 `counter level` 1~2（rank≥3 可为 2）
- 外来任务：`successChance − counterLevel×8`；发现检定提高
- **女间谍**（§30.5）：城级反间对女间谍 detect **+5**
- 任务结算时 roll 是否被捕；失败且发现 → 高概率被捕
- **月度清理**（`tickSpyMonth`，Session 52）：
  - 城已非本势力 → 驻守特工撤回 idle（冷却≥1）
  - **占城**时 `clearCityCounterOnCapture` 立即拆敌反间
  - 孤儿 `cityDefense`（特工缺失/非 counter_duty）删除
  - 灭亡势力（`isAlive=false`）特工 → `dead`，编制空出
  - home 城失守 → 迁往都城/任意己方城

### 29.6 损耗与外交

| 人身 | 处理 |
|------|------|
| 安全/负伤 | idle 或 recovering + cooldown |
| 被捕（男特工） | captive；双方友好 **−18** |
| 被捕（**女间谍**） | captive；双方友好 **−25**（§30.5 风险更高） |
| 死亡 | dead；编制空出 |
| 处决俘虏 | dead；友好再 −10 |
| 释放 | 回对方 idle；友好 +5 |
| 势力灭亡 | 全部特工 dead |

### 29.7 AI（对称）

每月：处置俘虏 → 边境/首都驻守反间 → 编制不足则招募 → **beautyStock≥4 且有空编制时训练女间谍** → 邻接敌城：有 detailed 情报时女间谍优先枕边风/离间，男特工破坏/刺杀。不作弊全知。

### 29.8 未做

多月潜伏、非邻接潜入、策反。  
已落地：服务端脱敏下发 ✅（§5.5）；献美点化女间谍 ✅。

### 29.9 RNG 与 AI 边界（当前实装）

- `spy.ts` 是共享结算层：普通密探招募、女间谍训练、献美点化、任务成功/识破/被捕/死亡，以及破坏、刺杀、枕边风、离间流言的结果均显式接收权威 `xorshift32-v1`。
- `spyAi.ts` 的 `Math.random()` 只负责 S15 AI 的“是否行动、目标与任务选择”，本轮按既定边界保留；一旦选定行动，招募/训练/任务结果仍调用共享结算层并使用回合注入的权威 RNG。
- 当前离间流言只实现“邻接第三势力降友好；无第三方则降目标城民忠”。**四面楚歌尚无运行时规则**，不得据此测试误报为已实现。
- `pnpm verify-plot-spy-rng` 覆盖普通招募、女间谍训练、献美点化、枕边风与离间流言的存读档确定性。

### 30.3 美女资源：获得（定稿数量）

| 方式 | 势力 beautyStock | 城 beautySeekLeft | 民忠 |
|------|------------------|-------------------|------|
| **寻访成功** | **+1** | **−1**（扣 1 次可寻） | 不降 |
| **寻访失败** | +0 | **不扣** | 不降 |
| **上贡/献美** ✅ | 己方−n / 对方+n | 不动 | 友好 **+12×n** |
| **外交转移** | 同献美 | 不动 | — |

| **计谋截获** | +1~3 | 视计谋 | 视计谋 |

**寻访（定稿）**

```
条件：己方城；beautySeekLeft ≥ 1；耗金（草案 50~80）
成功 → faction.beautyStock += 1
       city.beautySeekLeft  -= 1   // 「扣1」= 潜在可寻次数 −1
失败 → stock 不变；seekLeft 不变（可再试，只耗金）
// 不产生历史女角
```

**抢夺（定稿）**

```
来源 A — 攻占城池：
  gain = min(2+random(0~2), target.beautySeekLeft)  // 2~4，且不超过可寻
  attacker.beautyStock += gain
  target.beautySeekLeft -= gain
  target.stats.morale  -= 10~20

来源 B — 密探劫掠（可选）：
  gain = min(2~3, target.beautySeekLeft)
  同上；民忠 −8~15；暴露则外交−
```

原则：**寻访 = 势力+1 且该城可寻−1；抢夺 = 多拿 + 多扣可寻 + 伤民心。**

### 31.8 公式与上限

**L1 通用公式**（保留）：
- 成功率：基础45% + 女间谍(+20%) + detailed(+15%) − 反间level×8%。上限[10%, 85%]
- 识破率：基础20% + 女间谍(+10%) + 反间level×8%。上限[5%, 75%]
- 假情报/空城底线略抬（约 50~55%）

**L2 通用公式**：
- 基础成功率：35%（低于 L1，因为影响更大）
- 每完投一期（月）+5%
- 女间谍加成：+15%（L2 中女间谍可提升所有类型成功率，不限于美人计）
- detailed 情报加成：+10%
- 识破率：基础25% + 女间谍(+10%) + 反间level×10%。上限[10%, 85%]
- 识破后外交惩罚：友好−25~50（高于 L1 的 −15）
- 准备期：1~3 月不等。进度归零则计谋失败（可提前手动终止）

**L3**：无成功率/识破率。国策为确定性效果，切换冷却 6 月。

**并行上限**：
- L1 同时进行中：≤4 / 势力（不变）
- L2 同时进行中：≤2 / 势力
- L3 国策：1 / 势力

**RNG 与 AI 边界（当前 L1 实装）**：`plot.ts` 的 ID、成功、识破、目标/第三方选择、忠诚/友好降幅和女间谍被捕均使用回合注入的权威 `xorshift32-v1`。`plotAi.ts` 仅保留 S15 AI 是否施计、计谋种类与目标选择的独立 `Math.random()`；AI 一旦决定施计，创建 `Plot` 与后续月度结算均共用权威结算流。四类 L1 计谋与 S07 路径由 `pnpm verify-plot-spy-rng`（30/30）验证读档后完整结果与 draws 一致。

## 三十三、舌战系统

### 33.1 定位

舌战是文官版的单挑，用唇枪舌剑替代刀光剑影。核心场景：

- **劝降/登用** → 舌战说服，胜则忠诚大幅下降或直接加入
- **外交说服** → 舌战决定友好度提升倍率（成功 ×1.5，失败 ×0.5）
- **军师对垒** → 战斗中双方军师舌战，胜方计策成功率 +20%
- **王朗式骂战** → 纯羞辱，降对方阵营士气（历史事件代入）

### 33.2 论牌系统

双方每回合各选 **1 张论牌**，牌型互有克制：

| 论牌 | 攻值公式 | 克制对象 | 被克制 |
|:----:|----------|:--------:|:------:|
| **道理** | 政治/2 | — | **感情**（+50%伤害） |
| **感情** | 魅力/2 | 道理 | **典故**（+50%伤害） |
| **典故** | 谋略/2 | 感情 | **道理**（+50%伤害） |
| **诡辩** | 奇谋/2 | 无固定克制 | 破防时额外伤害 ×1.5 |

### 33.3 战斗流程

```
1. 初始化：双方论据 = 智力/2 + 谋略/4（底线 5）
2. 循环：
   a. 攻方选牌
   b. 伤害 = 攻方牌攻值
   c. 若攻方牌克制守方预判（AI）/ 牌型天生优势 → 伤害 +30%
   d. 诡辩若攻值 > 守方论据/2 → 破防，伤害 ×1.5
   e. 感情攻击额外削论据 ×10%（情绪打击）
3. 任一方论据 ≤ 0 → 败北
```

### 33.4 结果应用

| 场景 | 胜方 | 败方 |
|------|------|------|
| 劝降 | 目标忠诚 -40，可再劝 | 友好 -15，1 年内不可再劝 |
| 外交 | 友好倍率 ×1.5（上限 +30） | 友好倍率 ×0.5（下限 -10） |
| 军师对垒 | 己方计策成功率 +20% 本场 | 对方计策成功率 +10% |
| 骂战 | 对方阵营士气 -10 | 己方士气 -5 |

### 33.5 MVP 实现

- **无卡片动画**，仅文本显示：双方论据血条 + 选牌按钮 + 逐回合结算
- 后续优化：卡片插画 + 出牌动画 + 辩论语气配音

## 三十四、教育·科技·文化

> 三者形成闭环：**教育**育才→产出**文化**→解锁**科技**→科技反哺教育与经济。
> **儿童**（demographics.child）成丁时受城市教育值加成——现在的孩子决定未来的武将质量。

### 34.1 教育系统

每城可投入金发展教育（类似开发指令）：

```
教育值（0~1000）:
  投入金 80/季 → 教育值 +5~15
  若城中有武将"任教"（智力≥60 且当前无其他任务）→ 额外 +floor(教师智/50)×3
  城市文化 ≥ 200 时 → 教育效率 ×1.5
  太学设施（建设中）→ 教育值+5/季（被动）

儿童成丁时（child→adultMale/Female）:
  基础属性 = random（现有机制）
  教育附加值 = floor(城市教育值 / 100 × 出身倍率)
    出身倍率: 士族×1.5  宗室×1.2  豪强/平民/商贾×1.0  寒门×0.7
  任教武将智力/50 → 额外加成叠加

教育值还产出"文化":
  每季文化值 += floor(教育值 / 50)
```

**任教**：指派1名武将"任教"，该武将本季不能出征/开发/其他任务。任教武将自身智力缓慢成长（+1~3/年）。

### 34.2 科技系统

| **商业** | 商业产出+10% | +20%，贸易路线+1 | +30%，关税+20% |
| **军事** | 征兵质量+1 | 解锁特殊兵种 | 全兵种攻+10% |
| **城防** | 城墙耐久+10% | 守城伤害-15% | 三倍城墙（宫城外层） |
| **教化** | 教育效率+10% | 文人产出+50% | 全武将忠诚降速-50% |

**文化产出**（每季）：

## 三十五、货币·税收·俸禄

> **⚠️ 实装状态：纯设计，零代码实装**（Session 84 文档设计完成）。`Faction` 接口无 `coinQuality`/`salaryArrears`，`City` 无 `taxRate`，`turn.ts` 用旧产金公式，无俸禄发放引擎。记技术债 **D-0B-9**，独立 Session 实装（破坏性改动需先扩类型 + Zod + 全量 JSON + mask-state）。

### 35.1 货币成色

> 三国各有铸币，质量天差地别。金在账面上是"金"，但实际购买力取决于**货币成色**。

| 势力 | 史实币种 | 成色 | 含义 |
|:----:|:--------|:---:|------|
| **曹魏** | 五铢钱（续汉制） | **1.00** | 标准铜钱，购买力稳定 |
| **蜀汉** | 直百五铢 | **0.70** | 虚值大钱，含铜量仅面值1/100 |
| **东吴** | 大泉五百/大泉当千 | **0.50** | 虚值更甚，实质通胀严重 |
| **群雄** | 杂用旧钱 | 0.80~1.00 | 各州不一 |

**效果**：
```
实际购买力 = 账面积金 × 货币成色

例: 蜀汉支出500金买粮
  → 粮商实际收到 500×0.70 = 350金价值的铜钱
  → 同等金量下能买到的东西更少
```

**动态变化**（行政指令）：
```
铸币改革（耗金300，持续1年）→ 成色 +0.1
滥发虚值钱（战时应急）→ 成色 -0.1，立即获得金300
占产铜地（如荆襄铜矿）→ 成色下滑速度 -50%
```

### 35.2 税收系统

**改革当前自动产金公式**：
```
当前: goldProduced = commerce/9 × (0.7+adultShare) × goldMul
改为:
  毛收入 = commerce/9 × (0.7+adultShare) × goldMul
  税率   = 城市当前税率政策决定
  税收   = floor(毛收入 × 税率)
  金库收入 = floor(税收 × faction.coinQuality)  ← 成色折扣
  民间留存 = 毛收入 - 税收（影响民心）
```

| 政策（城市级） | 税率 | 民心 | 人口 | 说明 |
|:------------:|:----:|:----:|:----:|------|
| 轻徭薄赋 | 20% | +3/季 | +2%/年 | 长期发展 |
| 标准 | 30% | — | — | 默认 |
| 重税 | 50% | -5/季 | -2%/年 | 紧急扩军 |
| 战时征敛 | 70% | -10/季 | -5%/年 | 不可超过2季 |

**豪强税收隐匿**：
```
若城中任职武将有 tags 含 "豪强":
  基础税收 -15%（田产被隐匿）
  "检籍"指令（耗金200）→ 恢复全额，但该豪强忠诚-15
  "安抚"指令（耗金300）→ 恢复全额，持续3年
```

### 35.3 武将俸禄

按官位年俸，在每年1月统一发放：

| 官职 | 年俸（金） |
|:----:|:---------:|
| 无职 | 0 |
| 书吏/军候 | 20 |
| 县令/校尉 | 60 |
| 太守/将军 | 180 |
| 都督/大将军 | 500 |
| 丞相 | 800 |
| 君主 | 0 |

**欠俸后果**（每季检查）：
```
欠俸1季 → 忠诚 -2
欠俸2季 → 忠诚 -5，概率弹劾
欠俸3季 → 忠诚 -10，概率叛逃/投敌
```

**俸禄实际成本**：受势力货币成色影响。
```
势力支出 = 面值（不变）
武将实际感受 = 面值 × coinQuality
例: 蜀汉发180金给将军 → 武将实际觉得只值126金（180×0.7）
但忠诚扣除只看"是否足额发放面值"，不看实际购买力
→ 低成色势力养官更贵（同面值养不出同等忠诚）
```

## 三十七、总军师系统规则

> 总军师（Grand Strategist）是势力层的战略决策者。
> 详细职能与战役层作用见 `05-combat-system.md §十四`。
> 本节补充总军师在游戏规则层面的各项细则。

### 37.1 总军师 vs 参谋 vs 军师（官职）

| 角色 | 范围 | 人数 | 核心职能 | 出处 |
|:----:|:----:|:----:|:---------|:----:|
| **总军师** | 势力层 | 1 | 战略态势·献策·总军师对决 | §十四·本章 |
| **军参谋** | 战役层（每军） | 1参+1副参 | 陷阱·激励·斥候·撤退休整 | 05§13.6 |
| **军师（官职）** | 势力层 | 1 | 势力级军政建议（现有S11官职系统） | 04§九 |

**三者可兼任规则**：
```
总军师 ≠ 军师官职（可同一人但职责不同）
总军师 ≠ 军中参谋（不可兼任，一人不能同时定战略和当军参谋）
军师官职 + 总军师 → 可兼任（荀彧为曹操军师+总军师）
军师官职 + 军中参谋 → 不可兼任（军师是势力级，参谋是军级）
```

### 37.2 任命与解职规则

**任命条件**：
```
1. 智力 ≥ 90（初始）/ ≥ 85（功绩解锁后）
2. 与君主相性差 ≤ 50
3. 当前未被任命为军中参谋
4. 每势力限 1 人
5. 在君主所在城或都城
```

**解职条件**（任一触发）：
```
1. 君主主动解职（消耗政令次数 1）
2. 总军师被俘/死亡 → 自动空缺
3. 总军师忠诚 ≤ 50 → 概率自动辞职（每月判定 10%）
4. 总军师与君主相性差突变至 > 60（如君主更换）
```

**空缺处理**：
```
空缺时：
  - 势力保持上一季的策略不变
  - 无总军师加成（策略效果 ×0.5）
  - 无总军师对决能力
  - AI 势力空缺时自动从高智武将中选任
```

### 37.3 态势切换规则

| 规则 | 说明 |
|:----|------|
| **冷却** | 每季只能切换一次态势，切换后冷却 1 季 |
| **玩家操作** | 君主/总军师可在 LeftPanel 态势面板切换 |
| **AI 自动** | AI 势力总军师根据局势自动切换（每季判定，非玩家势力时） |
| **切换成本** | 免费（但冷却期内不可再切） |
| **影响范围** | 改变后即时生效，影响该势力所有 Army |

**AI 态势判定逻辑**：
```
offense ← 总兵力 > 相邻敌总兵力 × 1.5
defense ← 总兵力 < 相邻敌总兵力 × 0.6
development ← 城均金 < 500 且 无前线接敌
endurance ← 被围攻状态 或 总兵力 < 最大势力兵力 × 0.3
默认 ← 当前已持平
```

### 37.4 总军师对决详细规则

```
触发时机：每季度初自动判定
判定条件：双方势力均有总军师

对决效果：
  己方总军师智 - 敌方总军师智 ≥ 0 → 己方获得策略修正（用于§17自动战斗）
  己方 - 敌方 ≥ 15 → 识破敌方一次战役计谋
  己方 - 敌方 ≥ 30 → 反制：敌方计谋无效，己方获知敌方动向
  己方 < 敌方 → 无额外效果（但敌方获得对等优势）

战略计谋识破（S17 L2联动）：
  每项 S17 L2 计谋（釜底抽薪/调虎离山等）在战役层生效前，
  先经总军师对决判定是否被识破
  识破成功 → 该计谋本季不生效，消耗减半
  识破失败 → 正常生效

对决记录：
  每任总军师的识破/被识破次数影响其"战略总评"
  战略总评影响该势力外交信誉（高总评→更可信）
```

## 三十八、学派与信仰

> 城市的文化思潮——社会相信什么，比国家投入什么更能决定一个城市的性格。
> 学派是**被动加成系统**：建设施→倾向偏移→持续生效。偶尔有学派事件触发，不需要每回合操作。
> 与 §三十四 教育·科技·文化的区分：科技是国家金钱能买到的，学派是人民相信的。两者独立但互相反馈。

### 38.1 城市文化倾向

每个城市拥有 7 种学派倾向值（0~100），合计不封顶——城市可以儒道并立，也可以一家独大。

```typescript
interface CitySchoolInfluence {
  confucian: number;   // 儒
  daoist: number;      // 道
  buddhist: number;    // 佛
  mohist: number;      // 墨
  legalist: number;    // 法
  strategist: number;  // 纵横
  medical: number;     // 医
}
```

**决定因素**：

| 因素 | 影响方式 |
|:----|:---------|
| **初始值** | 见 §38.6 各势力初始倾向建议表 |
| **设施** | 建造文化设施 → 该倾向 +2~3/季 |
| **武将任教** | 指派有学派倾向的武将任教 → 可持续影响 |
| **邻城** | 邻城儒 ≥ 60 → 本城儒 +1/季（文化扩散） |
| **政策** | 势力级文化政策 → 全城倾向偏移 |
| **事件** | 学者来访/学派辩论 → 一次性大幅度变化 |
| **战争** | 长期战争 → 法+，儒-；被侵略 → 墨+ |

### 38.2 学派详表

#### 儒（Confucianism）— 教化·礼法·经学

> 东汉已是官方正统。董仲舒"罢黜百家"后形成的是儒法表里——以儒术缘饰吏治，吸收阴阳五行。
> 代表人物：孔融（孔子20世孙，北海修城立校）、郑玄（两汉经学集大成者）、卢植。

| 门槛 | 效果 |
|:---:|:-----|
| ≥ 40 | 该城文官（智≥70）内政指令效率 +10%（礼法约束） |
| ≥ 60 | 忠诚衰减 -20%/季；登用在野武将时相性匹配值 +10 |
| ≥ 80 | 解锁城政策「察举」：每季自动推荐 1 名在野儒士（智/政 ≥ 70） |

**与 §34 联动**：儒 ≥ 60 → 教化科技研发 +20%

#### 道（Daoism）— 太平·天师·无为

> 东汉朝两脉：老庄道家（哲学/隐逸）和道教（宗教化）。
> 太平道（张角·黄巾）和天师道（张鲁·五斗米道·汉中割据）。
> 东汉末民间疾苦，道教化运动正在兴起。

| 门槛 | 效果 |
|:---:|:-----|
| ≥ 30 | 人口自然增长率 +10%；施米效果 +15%（义舍传统） |
| ≥ 50 | 伤兵恢复速度 +25%/季（符水救治） |
| ≥ 70 | 战役层 Army 进入该城后疲劳 -20（义舍接待） |

**与 §34 联动**：道 ≥ 60 → 农耕科技研发 +15%

#### 佛（Buddhism）— 慈悲·译经·来世

> 汉明帝永平11年（68 CE）建白马寺于洛阳——中国第一座佛寺。
> 至东汉末仍很小众，集中在洛阳—徐州—下邳一线。笮融在徐州大修佛祠可容3000人。
> 《四十二章经》是最早汉译佛经。

| 门槛 | 效果 |
|:---:|:-----|
| ≥ 30 | 驻军士气恢复 +3/季（慈悲抚慰） |
| ≥ 50 | 与该城外势力外交初始好感 +5（佛缘）；解锁遣使西域选项 |
| ≥ 70 | 解锁特殊外交行动「遣使求法」→ 与西域势力关系 +20 |

**与 §34 联动**：佛主导时无直接科技加成（佛教不涉实务），但该城文化产出中有「译经」加成 +10/季。

#### 墨（Mohism）— 非攻·守城·机关

> 战国显学，汉初已衰微。但《墨子》城守各篇流传，工程/器械知识转入兵家和工匠体系。
> 墨家思想在东汉虽无组织化传承，但**守城之术**和**工匠精神**仍在。
> 本作设定：墨者工坊代表残存的墨家工程传统，非完整学派。

| 门槛 | 效果 |
|:---:|:-----|
| ≥ 40 | **§十五 设施建造速度 +20%**；城防耐久上限 +15%（墨子守城遗法） |
| ≥ 60 | **§十五 攻城器械耐久 +30%**；解锁城政策「墨者工坊」：可同时建造 2 件攻城器械（原限 1 件） |
| ≥ 80 | **§十七 自动战斗**攻城战伤害修正 +0.15（墨家器械专精） |

**与 §34 联动**：墨 ≥ 60 → 城防科技研发 +20%

#### 法（Legalism）— 律令·赏罚·集权

> 汉宣帝名言："汉家自有制度，本以霸王道杂之。"法家从未消失。
> 曹操的施政策略（唯才是举、重罚重赏）本质是法家集权路径。

| 门槛 | 效果 |
|:---:|:-----|
| ≥ 40 | 税收效率 +10%；征兵速度 +15% |
| ≥ 60 | 民心 -1/季（严刑峻法）；不欠俸时忠诚 ≥ 90 的武将永不投降 |
| ≥ 80 | 解锁城政策「以法治军」：该城征出的部队组织度 +10 |

**与 §34 联动**：法 ≥ 60 → 商业科技研发 +15%

#### 纵横（School of Diplomacy）— 合纵·连横·说辞

> 苏秦张仪的传统在三国时代并未消失——外交谋士的纵横捭阖在三国达到新高度。
> 代表人物：贾诩（说张绣降曹）、司马懿（早期外交斡旋）。

| 门槛 | 效果 |
|:---:|:-----|
| ≥ 30 | 外交报价权重 +10%；S17 L1 离间计成功率 +8% |
| ≥ 50 | S17 L2 秘密挖角成功率 +10%；S08 结盟友好 +5 |
| ≥ 70 | 解锁城政策「纵横捭阖」：与该城外交时，对方让步意愿 +15% |

**与 §34 联动**：无直接科技加成（纵横不是生产性学派）。

#### 医（Medicine）— 伤寒·青囊·坐堂

> 东汉医学高峰。张仲景《伤寒杂病论》、华佗《青囊书》（佚失）。
> 华佗游走于曹操/关羽之间，吴普/樊阿传承其术。

| 门槛 | 效果 |
|:---:|:-----|
| ≥ 30 | 瘟疫爆发概率 -30%（与 §23 伤病系统联动） |
| ≥ 50 | 伤兵恢复 +15%/季；该城武将体力年衰减 -5% |
| ≥ 70 | 该城武将受伤（§8.7 部位伤）恢复速度 ×1.5 |

**与 §34 联动**：医主导时无直接科技加成，但教育值有概率触发「医学院」事件。

| **道观** | 道 | 金 600，粮 500 | 2 季 | 道 +3/季，民心 +1/季 |

### 38.4 学派冲突

学派之间并非总和平共处——儒家与法家存在结构性冲突：

```
儒 ≥ 60 且 法 ≥ 40 同城：
  → 儒法冲突：民心 -2/季，税收效率儒法系数抵消
  解释：儒的宽仁（轻徭薄赋）与法的严苛（重税以养兵）相互抵触

佛 ≥ 40 且 道 ≥ 40：
  → 现状相安（东汉末佛教道教尚未争锋）
  → 佛 ≥ 60 → 道 -1/季（佛教自然吸引信众）
  → 道 ≥ 60 → 佛 +0（道为本土信仰，不受外来佛冲击）

墨不与其他学派冲突（已衰微，不争话语权）
医不与其他学派冲突（方技之学，不涉意识形态）
纵横不与其他学派冲突（权谋之术，不涉价值）
```

## 三十九、委任军团系统

> **设计状态**：本轮设计完成，引擎/UI 待实装。
> **核心定位**：后期势力城池众多时，玩家可将城池划区委任给 AI 都督自动管理，聚焦战略决策而非逐城内政操作。
> **职责边界**：委任都督管内政+军事（自动出征），外交权统一归君主。

### 39.1 委任区（DelegationRegion）

委任区是按**地域**划分的城池管理单元。每个委任区包含若干地理相邻的城池，委任一位都督全权管理。

**地域分区表**（基于东汉十三州格局 + 战略要地独立）：

| 委任区 | 覆盖范围 | 核心城池 |
|:------:|----------|----------|
| **中原** | 司隶·豫州·兖州 | 洛阳·长安·许昌·陈留·阳翟·平舆 |
| **河北** | 冀州·青州·并州 | 邺·平原·真定·晋阳·壶关 |
| **荆襄** | 荆州 | 襄阳·江陵·宛 |
| **江东** | 扬州 | 建业·吴·寿春·广陵 |
| **巴蜀** | 益州 | 成都·江州·汉中 |
| **关陇** | 凉州·司隶西 | 长安·冀县·姑臧 |
| **徐州** | 徐州 | 下邳·剧县 |
| **幽燕** | 幽州 | 蓟·涿·襄平 |
| **交广** | 交州 | 番禺·龙编 |
| **淮南** | 扬州北·豫州南 | 寿春·平舆（视势力边界可弹性归入中原/江东） |

> 0-A 30 城阶段可简化为 4~5 区（中原/河北/荆襄/江东/巴蜀），0-B 105 城时扩展至全 10 区。

**弹性规则**：
- 玩家可自由划城归属，不强制按上表分区（但系统按上表提供默认方案）
- 一个委任区最少 1 城，最多不超过上限（由官职爵位决定）
- 城池只能归属一个委任区
- 首都不可委任（始终由君主直辖）

### 39.2 委任都督（DelegationGovernor）

**任命条件**：

| 条件 | 要求 | 说明 |
|:----:|:----:|------|
| 官职 | ≥ 太守（单城）/ ≥ 都督（多城） | 太守仅可委任单城；都督可委任 2~4 城 |
| 属性 | 统 ≥ 70（影响出征效率）/ 政 ≥ 50（影响内政效率） | 不满足时效率 penalty |
| 忠诚 | ≥ 80 | 忠诚过低可被策反或独立 |
| 唯一性 | 一名武将同时只可担任一个委任区的都督 | 不可跨区兼职 |

**属性对委任效率的影响**：

| 都督属性 | 影响维度 |
|:--------:|----------|
| **统率** | 出征AI积极性 + 战斗胜率 + 征兵效率 |
| **智力** | 计策防御 + 识破敌计概率 + 内政开发效率 |
| **政治** | 内政开发效率 + 城民心稳定 + 税收效率 |
| **魅力** | 委任区内武将忠诚衰减减速 + 在野登用成功率 |

### 39.4 委任方针

都督的行动由玩家设定的方针驱动：

| 方针 | 内政权重 | 军事权重 | 征兵倾向 | 出征倾向 | 适用场景 |
|:----:|:--------:|:--------:|:--------:|:--------:|----------|
| **发展优先** | 70% | 30% | 低 | 低 | 后方新占城，需要稳定发展 |
| **军备优先** | 30% | 70% | 高 | 中 | 前线要塞，需屯兵备战 |
| **平衡型** | 50% | 50% | 中 | 中 | 常规委任区 |
| **攻略型** | 20% | 80% | 高 | 高 | 精锐军团，专攻敌境 |

**方针切换冷却**：每季可切换一次，切换后下季生效。

### 39.5 委任AI行为

委任都督每周（每回合）自动执行以下操作：

**内政（按方针权重分配金粮）**：
```
每回合执行顺序：
1. 检查委任区内各城是否有未完成的内政操作
2. 按方针分配可支配金粮：
   - 发展优先：农业 > 商业 > 城防 > 文化
   - 军备优先：城防 > 征兵 > 训练 > 农业
   - 平衡型：均衡分配
   - 攻略型：城防 > 征兵 > 训练 > 农业
3. 优先完成已有过半进度的开发
4. 各城保留最低粮草（守军 3 个月消耗）
```

**军事**：
```
每月执行：
1. 检查委任区边境城市是否有可出征的邻接敌城
2. 条件判断：
   - 兵力 > 敌城兵力 × 1.3（攻略型 × 1.0）
   - 粮草充足（出征部队 ≥ 3 个月消耗）
   - 出征武将忠诚 ≥ 80
3. 满足条件时自动组建出征部队（复用已有出征引擎）
4. 出征目标优先级：
   1) 委任区地图上邻接的敌城
   2) 敌兵力最弱的邻接城
   3) 有资源/战略价值的城（产马/产铁）
5. 胜率低于 30% 时不出征（攻略型不低于 20%）
```

**人事（可配置）**：
```
可选开启：
- 自动搜索委任区内各城在野武将（消耗金粮）
- 自动登用委任区所在城的在野武将
- 自动赏赐委任区内忠诚 < 70 的武将
```

**委任AI效率公式**：
```
委任效率 = 基础效率（60%）+ 都督统率/100 × 10% + 都督政治/100 × 10% + 方针系数（±10%）
基础效率 60% 保证即使都督平庸，委任区也不会完全停摆
```
