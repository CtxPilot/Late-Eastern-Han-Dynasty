# 设计一致性修订基线

> 状态：**已批准，分步实施**  
> 范围：跨 S01/S03/S04/S08/S09/S10/S15/S21 的规则治理，不新增并列大系统。  
> 原则：本文记录目标规则与迁移顺序；在代码、数据、UI、API 和验证全部完成前，不得把对应项标为“已实装”。

## 一、为什么修订

全量文档审查确认：游戏最有价值的主循环是“情报判断 → 任命与编成 → 郡域节点和补给 →
战斗模式选择 → 伤亡、人口、忠诚与外交后果”。当前主要风险不是缺系统，而是时间单位、
公式量纲、战斗层级、单挑交互、成长入口和文档摘要之间存在冲突，导致规则无法稳定平衡。

本轮冻结继续横向增加系统，先按下列 P0～P4 修复。每次只实施一个大系统或一组最小依赖，
遵守 `AGENTS.md` 的验证与文档双写规则。

## 二、批准后的统一规则

### P0 — 口径和真源

1. **时间真源**：`1 回合 = 1 个月`，`3 回合 = 1 季`，`12 回合 = 1 年`。
   “每月”是推进频率；“季初/季末”是第 1/4/7/10 月触发的季度结算，不再把“季”当作回合别名。
2. **战争结构**：统一称为“战争四层、三种交战结算模式”。
   四层是行政大地图 → 郡域战场 → 局部交战 → 单挑演出；三种结算模式是自动、标准、六角微操。
   白刃表现属于标准模式的表现层，不再与六角并列成第二套必经结算。
3. **规模数字**：阵型全量为 27（18 陆 + 9 水）；隐藏属性为 19（11 + 8）。
4. **原创表述**：允许并鼓励本项目原创程序化表现；禁止的是把虚构具名人物冒充史实，以及无授权仿制
   商业作品。历史剧本、演义层、假想剧本与纯玩法改编必须显示来源层级。

### P1 — 公式和契约

1. 所有概率公式统一以**百分点**书写，最终一次 `clamp` 到 `[5%, 90%]`；禁止小数概率和百分点混算。
2. 登用率目标公式：

   ```text
   40
   + (说客魅力 - 目标魅力) × 0.30
   + (1 - |相性差| / 150) × 40
   + 目标义理 × 2
   - 目标野心 × 3
   + 技能/关系/处境修正
   ```

3. 外交成功率目标公式：

   ```text
   35
   + 双边友好度 × 0.35
   + (我方声望 - 对方声望) / 100
   + 使者魅力 × 0.15
   + 正当理由/共同敌人/条约修正
   - 对方戒备与利益冲突
   ```

   双边友好度只能出现一次，不得同时“加友好”又“减目标友好”。
4. 同盟不再硬锁 10 年：基础期限 24 个月，期内可毁约但承担信誉、友好和宣战理由后果；
   联姻不硬锁 15 年，姻亲关系持续存在，60 个月内毁约追加更重惩罚。停战和互不侵犯使用明确月数。

### P2 — 决策密度和战斗公平

1. 单挑采用“**一次战略介入 + 自动演算**”：玩家在发起/应战时选择
   `强攻 / 持重 / 诱敌 / 委任` 之一，仅改变自动指令权重和风险，不逐回合点牌。
   AI 与跳过演出均走同一权威结算。文案不得把玩家不可干预的内部随机称作玩家“心理博弈”。
2. 吕布保持全游戏最强单挑者，但不拥有数学上的绝对胜利：
   - 所有必杀仍服从核心克制，只给吕布抗克制和伤害优势；
   - 吕布可被击败；传奇保护只把首次致死/被俘结果改为重伤撤退，不篡改胜负；
   - 禁止“不可化解 + 被动高概率化解 + 失败减伤 + 不死不俘”四重叠加。
3. 成长界面收敛为四个玩家可读入口：人物成长、军团战备、阵型精通、战役态势。
   其余底层数值作为解释这些入口的细项，不再同时抢占主 UI 和奖励反馈。

### P3 — 经济、内政与 AI

1. 持续型内政是目标规则；当前即时开发明确标为 Demo 兼容切片。迁移时必须提供项目队列、
   剩余月数、中断损失和季度收益预览。
2. 建立标准化年度预算表：以单城、三城和十城三个样本核对金粮收入、军费、俸禄、
   建设回收期、战争损失和行政成本；没有预算表不得继续增加资源种类或常驻消耗。
3. AI 从“每势力一支进攻军”迁移到受资源、将领和前线数量约束的多线计划；
   难度优先提高信息处理、编成和目标选择，经济修正设软上限，不再给武将五维永久 `+5`。
4. 增加反滚雪球支点：跨城行政成本、占领整合期、补给线压力与弱势外交结盟价值。

### P4 — S09 语义迁移

S09 目标名称改为“**宫廷人脉**”（旧代码字段 `beautyStock/beautySeekLeft` 暂保留以便迁移）。
资源代表可用于赏赐、联姻牵线、谍报掩护和宫廷交涉的人脉机会，不代表人口中的女性数量：

- 删除“约 400 成年女性 = 1 点”“寻访扣成年女性”的规则；
- 来源改为市井声望、豪族往来、宴饮、战后接管与事件；
- 历史女角仍只通过史实事件、亲属跟随、联姻和剧本配置出现；
- 迁移完成前，UI 必须避免把库存描述成可买卖或抢夺的“女性实体”。
