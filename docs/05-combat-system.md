## 一、战场模型

### 1.1 六角网格 (Hex Grid)

```
战场为六角网格(Hexagonal Grid)
  - 默认战斗地图：20列 × 15行
  - 大型战役：30列 × 20行
  - 攻城战：特定地图(城+野外)

坐标系统(Axial Coordinates)：
  axis q: 0 → right
  axis r: 0 → down-right

移动方向(6方向)：
      NW    NE
        \  /
    W -- 〇 -- E
        /  \
      SW    SE
```

### 1.2 移动范围计算

```
移动范围 = 部队机动 m
BFS/洪水填充算法，从当前位置出发

基础公式：
  移动力消耗 = 兵种基础移动 + 阵型修正 + 地形修正 + 天气修正

特殊规则：
  - 不可穿越敌方部队占据的格子
  - 友军可穿过(不消耗额外移动)
  - 水面仅水军可自由移动，其他兵种需桥/船
```

```

  weatherChangeTimer -= 1
  if weatherChangeTimer ≤ 0:

```

### 5.5 主副将与参谋编成系统

> 一支部队由 **1 名主将** + **N 名副将** + **0~1 名参谋** + **0~1 名副参谋** 组成。
> 副将带兵独立作战，参谋/副参谋为幕僚不带兵，提供智略加成。
> 编队精神参考经典策略游戏，但规则为本项目独立设计。

#### 5.5.1 编成规则

**编成权限**由主将的**武官官职**决定基础，**功绩等级**和**爵位**提供叠加（受上限约束）：

| 武职 | 副将 | 参谋 | 副参谋 | +功绩(副将) | +爵位(总人数) | **上限** |
|:----:|:----:|:----:|:------:|:----------:|:------------:|:-------:|
| 军候 | 0 | 0 | 0 | — | — | **1** |
| 校尉 | 1~2 | 0 | 0 | +1(Lv5) | +1(侯) | **4** |
| 将军 | 2~3 | 1 | 0 | +1(Lv10) | +2(县侯+) | **7** |
| 大将军 | 4~5 | 1 | 1 | +1(Lv15) | +1(侯) | **9** |
| 君主 | 5 | 1 | 1 | — | +2(王/帝) | **10** |

**副将编成条件**：
- 主将与副将在同一城市
- 副将体力 ≥ 60%
- 主将与副将关系非嫌恶以下（pairAffinity ≥ -60）
- 每个副将只能编入一支部队

**参谋编成条件**：
- 与主将在同一城市
- 智力 ≥ 85
- 不要求任何官职（独立于势力军师任命）
- 不带兵，无独立 Squad

**兵种配置**：
- 主将决定部队的**主兵种**（用于阵型判定·克制关系·战场图标）
- 每个副将可带**自己的兵种**（可与主将不同）
- 阵型的兵种限制仅约束主将兵种，副将兵种受阵型**通用规则**影响（§5.5.3）

#### 5.5.2 Squad 系统

一支部队在战场上拆分为多个 Squad（小队），每个 Squad 对应一名武将及其直属兵力。参谋/副参谋**不带兵，不设 Squad**，以幕僚身份附于主将帐下。

```typescript
interface BattleSquad {
  officerId: number;
  unitType: UnitType;
  troopCount: number;
  maxTroops: number;
  morale: number;
  isMain: boolean;
  position: 'center' | 'left' | 'right' | 'rear';
}
```

**Squad 行为规则**：
```
主将 Squad（isMain=true）:
  - 决定部队的战场图标/主兵种/阵型适配
  - 被击破 → 整支部队溃散（副将可接替，士气-40%）
  - 主将专属技能/特性以 100% 效力作用于全队

副将 Squad（isMain=false）:
  - 独立参与攻防计算（汇总至部队总输出）
  - 独立承受伤害（副将 Squad 可先于主将溃散）
  - 副将阵亡 → 该 Squad 消失，部队总战力下降
  - 副将专属技能/特性以 50% 效力作用于全队（仅被动类）
  - 副将接替主将 → 该 Squad 升格为主将（士气惩罚减半，若关系友好以上）

参谋（无 Squad）:
  - 不带兵，不承受伤害
  - 被动加成（计策/视野）常驻全队
  - 主动策略消耗参谋体力
  - 主将溃散/被俘时随主将命运（按智判定逃脱或被俘）
```

**战斗表现公式**：
```
部队总攻 = 主将攻 + Σ(副将攻 × 0.3 × 关系系数)
部队总防 = 主将防 + Σ(副将防 × 0.3 × 关系系数)
士气衰减速度 = 基础 × (1 - 平均关系系数 × 0.2)
    关系系数: 义兄弟=1.0, 父子/兄弟=0.8, 亲密=0.5, 友好=0.2,
              普通=0.0, 嫌恶=-0.3（不可同队）
```

#### 5.5.8 参谋系统

参谋是部队中的智囊角色，不带兵，以幕僚身份提供智略支持。

**编成规则**：
- 独立槽位，不占副将名额
- 智力 ≥ 85
- 不要求军师/任何官职
- 每支部队限 1 名参谋（大将军/君主可额外设 1 名副参谋）

**战场作用**：

| 类型 | 效果 | 公式 |
|------|------|------|
| **火计加成** | 火计威力/成功率提升 | `+ 参谋智 / 50 %` |
| **战法触发** | 战法额外效果触发率提升 | `+ 参谋智 / 100` |
| **计策识破** | 敌方计策识破率 | `+ 参谋智 / 100` |
| **视野** | 部队视野半径 +1 格 | 常驻 |
| **反迷雾** | 可查看相邻敌方兵种/阵型/兵力档位 | 常驻 |

**主动策略（消耗参谋体力）**：

| 策略 | 条件 | 消耗 | 效果 |
|------|:----:|:----:|------|
| **激励** | 体力 ≥ 30 | 体力 -15 | 全队士气 +15，不可叠加 |
| **陷阱** | 智力 ≥ 90 + 体力 ≥ 40 | 体力 -20 | 当前位置布设简易陷阱，敌方进入受 `智×1.5` 伤害 |
| **撤退休整** | 体力 ≥ 20 | 体力 -10 | 本回合有序撤退，不触发溃散判定 |

**生存机制**：
- 不带兵，不设独立 Squad，不独立承受伤害
- 主将溃散/被俘时参谋自动判定：智力 ≥ 80 → 逃脱，否则被俘
- 副参谋在参谋阵亡后可升格接替（效果减半至战斗结束）

**爵位精简等级（7 级）**：

| 亭侯 | +1 | 最低等列侯 |
| 乡侯 | +1 | 中等列侯 |
| 县侯 | +2 | 最高等列侯 |

### 5.6 部队品质系统（经验·组织度·士气）

> 部队品质由三个独立维度构成：**经验等级**决定战斗力上限、**组织度**决定阵型执行力、**士气**决定临场发挥。
> 三者独立运算，叠加作用于部队表现。
> 经验与组织度在 §25（04-game-systems.md）有设计基础，本节为统一实装规范。

#### 5.6.1 经验等级（Lv1~7，沿用 §25 设计）

| Lv | 名称 | 所需经验 | 攻击 | 防御 | 士气上限 | 特殊 |
|:--:|:----:|:--------:|:---:|:---:|::-------:|------|
| 1 | 新卒 | 0 | — | — | 100 | 默认 |
| 2 | 训练兵 | 100 | +5% | +5% | 105 | — |
| 3 | 老兵 | 300 | +10% | +10% | 110 | — |
| 4 | 劲旅 | 600 | +15% | +15% | 115 | 暴击+2% |
| 5 | 精锐 | 1000 | +20% | +20% | 120 | 暴击+4% |
| 6 | 百战 | 1800 | +25% | +25% | 125 | 士气≥50时不溃逃 |
| 7 | 铁军 | 3000 | +30% | +30% | 130 | 全歼保留30%经验 |

**经验获取**：
```
参战存活: +20xp（每场战斗基础）
击破敌方一队: +30xp（每队）
主将被斩/俘: +50xp（对面）
以少胜多: ×1.3倍
胜仗存活: ×1.5倍
全灭: 经验归零（Lv7铁军保留30%）

```

**补员稀释**：
```
新经验 = (旧兵数×旧xp + 新兵数×0) / (旧兵数+新兵数)
即: 1000老兵 + 500新兵 → (1000×xp + 500×0) / 1500 = 原有经验的2/3
```

**训练等效**：城市执行"训练"指令时，被训部队获得0.3次等效战斗次数/季（§19.2）。

#### 5.7.2 部曲数据结构

```typescript
interface Officer {
  // ...现有字段
  personalTroops: number;         // 部曲当前人数
  personalTroopExp: number;       // 部曲经验值（独立于部队经验）
  personalTroopType?: UnitType;   // 部曲兵种（留空则与主将兵种一致）
```

#### 5.7.3 部曲编成规则

```
编队时:
  部队初始兵力 = officer.personalTroops + 募兵/征召兵

  初始经验 = weightedAvg(officer.personalTroopExp, 募兵经验=0)
    例: 许褚800部曲(xp=600) + 200新兵(xp=0) → (800×600+0)/1000 = 480xp(Lv4劲旅)

  部曲兵种: 若personalTroopType有值则强制为该兵种（高顺=陷阵营）
            否则随主将当前统领兵种
```

**部曲随人走**：
```
调任: 部曲全部带走（城市兵力不变，只调动个人）
被俘/释放: 部曲随主将消失/恢复
投降/登用: 部曲跟随新势力（私兵只认主将，不认势力）
```

#### 5.7.4 部曲成长与损耗

```
战后存活:
  存活比例 = 战后部队总兵力 / 战前部队总兵力
  officer.personalTroops = 战前personalTroops × 存活比例（向下取整）
  例: 许褚800部曲参战，伤亡50% → personalTroops = 400

经验成长:
  officer.personalTroopExp = 战后部队经验（同步更新）
  部曲经验与部队经验双向绑定——部曲是部队的核心种子

补充:
  部曲不可直接补充（无"征兵+部曲"选项）
```

### 5.8 军屯田与家属制度

#### 5.8.1 军屯田

驻军在非战时可选屯田自给，与训练并行（训练效果减半）。

**机制**：
```
军屯田状态可切换（每季可操作一次）:
  关 → 正常训练/备战
  开 → 部队开始屯田

产粮公式:
  foodProduced = floor(troops × (city.stats.farm / 100) × seasonMul × 0.5)
  seasonMul: 春×0.8, 夏×1.0, 秋×1.5, 冬×0.3

代价:
  troopsMorale: -3/季（士兵厌耕）
  训练效果: 减半（经验获取×0.5，取代原来×1.0）
  组织度: -2/季（耕作荒废阵列）

可开屯田条件:
  城市非战斗中
  troops > 0
  非出征状态
```

**适用场景**：长期对峙时减少粮耗压力（如诸葛亮屯田渭南）。

#### 5.8.2 家属制度（质任·溃败）

**核心逻辑**：每名士兵的家属 = 征兵城市的 `demographics` 人口。

```
征兵时:
  士兵来源 = city.demographics.adultMale
  家属绑定 = 同城的 adultFemale(妻) + child(子女) + elder(父母)
  存入 city.garrisonFamilies: number（家属人口数）
```

**默认：家属在征兵城**

**质任制（曹魏政策）——迁移家属**：
```

消耗: 金500
效果: 将该城全部士兵家属迁往指定后方城市（通常是首都）
       city.garrisonFamilies → 目标城
意义: 前线部队士气稳定，但首都若陷落则全国崩盘
```

**家属所在城市被攻占时的效果**：

### 6.1 基础伤害公式

```
damage = (攻击力 × 克制系数 × 阵型加成 × 地形修正 × 兵力系数) - 敌方防御力

详细展开：
  baseAttack = 兵种基础攻击 + 武将武/10 + 宝物攻击加成 + 技能加成
  baseDefense = 兵种基础防御 + 武将统/10 + 宝物防御加成 + 技能加成
  
  finalAttack = baseAttack
    × 兵种克制系数(0.6 ~ 1.5)
    × 阵型攻击修正(0.8 ~ 1.2，含阵型)
    × 地形攻击修正(0.7 ~ 1.0)
    × 部队剩余兵力系数(0.3 + 0.7*currentTroops/maxTroops)
    × 士气系数(0.6 + 0.4*morale/100)
  
  finalDefense = baseDefense
    × 阵型防御修正
    × 地形防御修正
  
  rawDamage = finalAttack - finalDefense
  damaged = max(1, round(rawDamage * random(0.9 ~ 1.1)))
```

完整战斗事件流（含暴击/反击/连击判定时序）见 **§6.5**。

> **引擎状态（Session 89）**：§6.2~§6.5 暴击/反击/连击引擎**最小切片已实装**（`server/src/battle/crit.ts` + `attackUnit`/`simpleAi.doAttack` 接入 `resolveAttack`）。暴击率/倍率、反击触发/系数、连击率/衰减、防循环规则、阵型修正表(§4.2)、通用技能联动、专属(武圣/无双/刚烈/恶来/龙胆/咆哮/天义/虎痴/骑神/神将)全部落地。简化项：武将特性用 skills proxy 近似(0-A officers.json 无 traits)；宝物按 officer.id 签名映射(0-A 无装备系统)。

### 6.3 反击

**触发条件：**
```
1. 被近战攻击（相邻六角格含冲锋），被攻击方存活
2. 被攻击方有能力反击
3. 攻击方在反击方的攻击范围内（近战=相邻格，长枪兵=相邻+1格）
4. 每对攻防仅1次反击
```

**不可反击情形：**
```
弓箭手 / 弩兵（被贴身时）
混乱 / 眩晕状态
士气 ≤ 30
已行动完的部队（本回合已攻击过）
```

**反击伤害公式：**
```
反击伤害 = 正常伤害 × 反击系数

通用反击系数 = 0.6
  + 技能修正（固守Lv1~5：+0.05~+0.25）
  + 特性修正（铁壁Lv1~5：+0.03~+0.15）
```

**反击系数表（兵种基础）：**
```

长枪兵           ×0.5（击退效果不作用于反击）
骑射手           ×0.4
水军             ×0.6
弓兵/弩兵        —（不可反击）
```

**反击暴击：**
```

暴击率 = 基础暴击率 + 5%（反击额外加成）
```

### 6.4 连击

**定义**：一次攻击（含反击）结束后，有概率再追加一次攻击。

**触发条件：**
```
1. 攻击方本回合未触发过连击（每回合最多1次连击）
2. 目标仍存活
3. 目标仍在攻击范围内
4. 非弓箭手远程攻击（近战/骑射可）
```

**连击率公式：**
```
chainRate = 基础10%
  + 武力/100                  (吕布100→+1%)
  + 兵种修正                  (轻骑兵+5%, 轻步兵+3%)
  + 适性修正                  (S+5%, A+3%)
  + 阵型修正                  (锥形+5%, 锋矢+3%, 车悬+5%)
  + 技能修正                  (见下方§6.4.1)
  + 特性修正                  (见下方§6.4.2)
  + 专属修正                  (见下方§6.4.3)
  + 宝物修正                  (丈八蛇矛+5%, 双铁戟+8%)
  - 士气<50: -5%
  - 体力<40%: -5%

上限 40%，下限 3%
```

**连击伤害：**
```
连击伤害 = 正常伤害 × 0.6
  暴击时 = 正常伤害 × 0.6 × 暴击倍率
  吕布·无双：连击不衰减 → 连击伤害×1.0（取代×0.6）
```

**连击暴击：**
```
连击独立进行暴击判定
暴击率 = 基础暴击率 × 0.7（连击衰减）
  技能/特性/专属的暴击率加成仍有效（先加后×0.7）
  无双：不衰减 → ×1.0（取代×0.7）
```

**连击限制：**
```
- 每回合仅触发1次（反击触发的连击也算）
- 连击不再触发连击（防止无限循环）
- 连击不再触发反击（被攻击方已在本轮反击过）
- 连击消耗：额外气力 -5
```

### 6.5 三者联动事件流

```
[攻方A] 攻击 [守方B]
  │
  ├─❶ 暴击判定（独立roll）
  │    ├─ 基础暴击率（§6.2）
  │    ├─ +技能加成（骑术/弓术/勇武/急攻）
  │    ├─ +特性加成（刚力/猛进/先登/乱战/夜战）
  │    ├─ +专属加成（武圣/无双/虎痴/骑神/神将）
  │    └─ → 暴击：伤害×暴击倍率（含刚力/武圣加成）
  │
  ├─ 伤害结算（§6.1 full formula）
  │    └─ 兵力/克制/地形/阵型/士气/宝物
  │
  ├─➋ 反击阶段（B存活 + 可反击）
  │    ├─ 技能检查：沉着Lv≥3 → 混乱不失去反击
  │    ├─ 特性检查：铁壁/镇守/乱战 → 加成反击率与伤害
  │    ├─ 专属检查：刚烈→必反+必暴 / 恶来→可反2次
  │    ├─ 反击伤害 = 正常伤害 × 反击系数
  │    ├─ 反击暴击判定（独立roll + ∀技能/特性/专属加成）
  │    └─ 反击后 → ➍ 反击后连击判定
  │
  ├─➌ 连击阶段（A的攻击后触发）
  │    ├─ 连击率判定（技能+特性+专属）
  │    ├─ 连击伤害 = 正常伤害 × 连击系数（基础0.6）
  │    ├─ 连击暴击判定（独立roll ×0.7衰减）
  │    └─ 连击不再触发→反击/连击（防循环）
  │
  └─➍ 反击后连击阶段（B的反击可触发连击）
       └─ 规则同➌，但A为被反击方的连击
          └─ 连击不再触发→反击/连击（防循环）
```

**防循环规则总表：**

| 事件 | 可触发暴击 | 可触发连击 | 可触发反击 |
|:----:|:----------:|:----------:|:----------:|
| 普攻 | ✅ | ✅ | —（攻击方不防） |
| 反击 | ✅ | ✅ | ❌（已反击过） |
| 连击 | ✅（×0.7） | ❌ | ❌ |
| 战法/计策 | ❌（按specialEffect） | ❌ | ❌ |

## 八、单挑系统

> 单挑拥有**完全独立的规则体系**（暴击/反击/连击/伤害/技能），与战场部队系统（§6）互不干扰。
> 引擎状态：`[~]` **最小切片已实装（Session 88）**——§8.5~§8.11 全自动结算引擎 + DuelPanel UI + API 可玩；宿命对决(§8.14.1)/大会(§8.17)/战后伤势延续(§8.7.3) 后置。

### 8.1 设计总览

单挑是武将之间的1v1决斗，采用**全自动结算**模式（发扬自**经典三国策略游戏**）——玩家不需要选择指令，一切由引擎根据武将属性/技能/武器自动计算，通过叙事文本和HP动画呈现。

引擎内部使用**7指令体系（核心三角 + 辅助三选 + 暗袭）× 经典式三向克制 + 武器分化 + 部位受伤**的深化系统进行逐回合推演，但玩家只看到结果，不参与操作。

单挑流程是一个有限状态机，完整生命周期如下：

```
  [战场回合中] → 触发/发起 → PRE_DUEL(阵前对话+射箭) → DUELING(多回合循环)
      ↑                                                         │
      └────────────── RESOLVED(结算影响) ← RESOLVING(结局判定) ←┘
```

- 单挑期间**战场暂停**，单挑结算完毕后继续战场回合
- 每场战斗每方每回合最多发起 **1次** 单挑
- 单挑不消耗战场行动次数，但消耗发起者 **20 气力**

### 8.5 单挑回合（DUELING 主循环 — 引擎内部逻辑）

> 以下所有指令选择、克制判定、伤害计算均为**引擎内部自动推演**，玩家不可见、不可操作。

> 这就是**经典三国策略游戏**的单挑体验——你不选指令，你**看**单挑。

#### 8.5.1 七指令体系

> 设计发扬自**经典三国策略游戏**的三向克制精神：核心三角（猛攻·牵制·必杀）互相克制形成心理战；辅助三选（格挡·闪避·周旋）提供多元战术；暗袭为一次性底牌。**必杀不再游离于克制循环之外，而是循环的主角。**

##### 核心三角（攻击型，参与三向克制）

| # | 指令 | 伤害修正 | 命中修正 | 防御修正 | 消耗 | 克 | 被克 |
|:--:|------|:--------:|:--------:|:--------:|:----:|:--:|:----:|
| 1 | **猛攻** | +30% | +0% | **-20%** | — | **牵制** | 必杀 |
| 2 | **牵制** | +0% | +15% | +0% | — | **必杀** | 猛攻 |
| 3 | **必杀** | **MAX** | **100%** | -30% | 50气力 | **猛攻** | 牵制 |

> **克制效果**：克方 +30% 伤害，被克方命中 -15%（必杀克猛攻时，猛攻方防御 -20% 叠加生效）。
> **牵制克必杀**：牵制方化解必杀，必杀伤害 **-50%**（半减），牵制方伤害按正常计算。这是整场单挑最戏剧性的时刻——必杀被破，攻守逆转。

##### 辅助三选（防御/牵制型，独立交互）

| # | 指令 | 类型 | 效果 | 限制 | 克 | 被克 |
|:--:|------|:----:|------|:----:|:--:|:----:|
| 4 | **格挡** | 防御 | 所有伤害 -50%；30% 概率反手一击（伤害×0.5） | — | — | 周旋 |
| 5 | **闪避** | 防御 | 免疫猛攻/牵制伤害；下回合自身命中 +20% | — | — | 周旋、必杀 |
| 6 | **周旋** | 攻击(辅助) | 命中 +20% | — | 闪避、格挡 | 猛攻 |

> **周旋克格挡**：格挡反手失效（被周旋打乱节奏）。
> **周旋克闪避**：闪避免疫失效 + 周旋方 +30% 伤害。
> **必杀无视闪避**：必杀必中，穿透闪避。闪避对必杀无效。
> **猛攻克周旋**：猛攻属于核心三角，对辅助指令周旋保有克制（+30% 伤害）。

##### 一次性底牌

| # | 指令 | 类型 | 伤害 | 特殊 | 限制 |
|:--:|------|:----:|:----:|------|:----:|
| 7 | **暗袭** | 特殊 | 武 × 武器系数 | 无视防御；对手下回合命中 -30% | 每场 1次；不耗气力；副武器专用 |

#### 8.5.2 克制关系图

**核心三角（发扬自经典三国策略游戏）：**
```
           猛攻
          ↗   ↘
   必杀 ←         → 牵制
          ↘     ↗
            (必杀克猛攻·牵制克必杀·猛攻克牵制)
```

**全克制链总览：**
```
猛攻 ──克──→ 牵制 ──克──→ 必杀 ──克──→ 猛攻（核心三角）
猛攻 ──克──→ 周旋
周旋 ──克──→ 闪避、格挡
必杀 ──无视──→ 闪避（必中穿闪）
```

**博弈心理（单挑的核心乐趣）：**
- 对方可能放必杀 → 你出牵制化解（牵制克必杀）→ 最高回报
- 你出牵制 → 对方怕被克而出猛攻 → 你被猛攻克
- 你出猛攻压制 → 对方出必杀一击致命 → 你被必杀克
- 这就是经典三向心理战的核心——没有安全牌，每回合都在赌命

#### 8.5.3 每回合结算流程

```
1. 先手方选指令 → 后手方选指令
2. 命中判定：
   baseHitRate = 80% + 敏捷差/10 + 武器命中修正 + 指令命中修正 + 闪避蓄力(+20%)
   必杀: 100% 必定命中（穿透闪避）
   暗袭: 95%
3. 克制判定（核心三角 + 辅助链）：
   若猛攻 vs 牵制 → 猛攻方 +30%伤害，牵制方命中-15%
   若牵制 vs 必杀 → 牵制方化解！必杀伤害 -50%，牵制方伤害正常
   若必杀 vs 猛攻 → 必杀 MAX + 必中，猛攻方防御-20%已含
   若猛攻 vs 周旋 → 猛攻方 +30%伤害
   若周旋 vs 闪避 → 闪避免疫失效，周旋方 +30%伤害
   若周旋 vs 格挡 → 格挡反手失效
   若必杀 vs 闪避 → 必杀无视闪避，必中
   若无克制 → 各取基础值
4. 伤害计算（见§8.6）
5. 暴击判定（见§8.11）
6. 格挡反击判定（仅格挡指令 + 未被周旋克制时，见§8.11）
7. 连击判定（见§8.11）
8. 受伤判定：每损失30%体力 → 1次部位受伤判定（见§8.7）
9. HP更新 → 进入下一回合 / 一方HP≤0 → 进入RESOLVING
```

### 8.6 伤害公式

#### 8.6.1 基础伤害

```
baseDamage = |进攻方武力 - 防守方武力| × weaponPower × (1 + 指令修正)
  weaponPower: 武器基础威力（见§8.9）
  指令修正: 猛攻+30%, 牵制+0%, 必杀+100%, 周旋+0%

最终伤害 = baseDamage
  × 随机浮动(0.90 ~ 1.10)
  × 体力系数(0.5 + 0.5 × currentHP/maxHP)
  × 专属修正(如吕布: +25%)
  - 格挡减伤(指令为格挡时: ×0.5)
  - 牵制化解(牵制克必杀时: 必杀伤害×0.5)
  - 闪避免疫(闪避对猛攻/牵制: 伤害=0; 必杀无视闪避)

保底伤害 = 1
暗袭伤害 = 武 × weaponPower × 随机浮动(0.90~1.10)（无视防御，无视格挡减伤）
```

#### 8.6.2 隐藏属性集成（参照 §32.4，04-game-systems.md）

> 8项隐藏属性（1~100，勇猛除外为0~7）各自独立影响单挑的不同维度。
> 以下为全量公式速查。具体数值参见 officers.json `hidden` 字段。

| 属性 | 范围 | 在单挑中的作用 | 影响比重 |
|:----:|:----:|--------------|:--------:|
| **力量** power | 1~100 | 重击伤害附加：`power × weaponPower × 0.1`；猛攻额外伤害 +`power/50`% | 中 |
| **爆发** burst | 1~100 | 先手率 +`burst/10`%；必杀率 +`burst/4`；连击触发率 +`burst/50`% | 高 |
| **敏捷** agility | 1~100 | 命中率 +`agility/10`%；闪避后下回合命中额外 +`agility/20`%；反击率 +`agility/100`% | 高 |
| **运气** luck | 1~100 | 暴击率 +`luck/5`%；受伤概率 -`luck/20`%；被斩概率 -`luck/50`% | 高 |
| **谋略** strategy | 1~100 | 预判对方指令概率 +`strategy/50`%（读AI模式）；牵制克必杀时化解率额外 +`strategy/100`%；格挡反手触发率 +`strategy/200`% | 中 |
| **奇谋** tactics | 1~100 | 暗袭伤害 +`tactics/20`%；首回合出手概率 +`tactics/50`%（不按常规出牌）；暴击系数 +`tactics/200` | 低 |

| **勇猛** valor | 0~7 | 基础单挑触发率；被斩概率 -`valor×2`%；必杀权重 +`valor×2`% | 高 |

> **勇猛与力量的区别**：勇猛(0~7)是战斗意志/胆量，决定敢不敢动手、敢不敢拼命；力量(1~100)是纯物理输出能力。
> 高勇猛+低力量 = 莽夫（张飞早期）；高力量+低勇猛 = 冷静杀手（典韦）。

> 隐藏属性中有六项碾压级数值，只有谋略30是明牌弱点——但纸面上的"弱点"在绝对力量面前几乎不成立。

**核心打法 — 绝对碾压：**
```
吕布的必杀 = 不可化解（天下无双 → 牵制克必杀对吕布无效）

吕布防御必杀 = 被动·高概率化解（无需选择牵制指令）：
  化解概率 75%（基础50% + 天下无双+25%）
  化解效果：必杀伤害 -70%（高于常人 -50%）
  即使化解失败(25%)：必杀伤害仍 -20%（威压气场削弱）

→ 核心三角在吕布身上只有两条边：
   吕布必杀 ─(不可化解)─→ 对方猛攻
   对方牵制 ─(无用)────→ 吕布必杀
→ 吕布没有天敌。核心三角对他而言是单向碾压。
```

**"无敌"的数学基础：**
```
吕布普攻猛攻伤害(武100, 力量98, 刀1.2):
  base = |100 - 对方武| × 1.2 × 1.3 = 156+ (假设对方武0)
  力量附加: 98 × 1.2 × 0.1 = 11.76
  对武88(华雄): base=12×1.2×1.3=18.72 + 力量11.76 = 30.48
  浮动±10% → 27~34 普攻伤害（等于别人的必杀）

吕布必杀伤害(三连击起手):
  base = 12×1.2×2.0(必杀) = 28.8 + 力量11.76 = 40.56
  三连击: 第一击40.56 + 第二击28.4(=40.56×0.7) + 第三击20.3(=40.56×0.5)
  合计 ≈ 89 (独立暴击概率各36% — 三发全暴击=178+)
  → 一套带走任何武将
```

**AI行为（极度危险且不可预判）：**
```
必杀权重: +50%（天下无双不需要理由）
猛攻权重: +20%（偶尔玩玩）
牵制权重: +0%（不屑化解别人——他有75%被动化解率，犯不着主动牵制）
格挡权重: +0%（天下无双不防御）
闪避权重: -5%（偶尔意思一下）
周旋权重: -5%（戏耍对手）
暗袭权重: +5%（方天画戟暗袭可用时）

吕布不需要预判——因为没有人能威胁他。
对方的一切谋划在绝对力量面前毫无意义。
```

**"无敌"的真正含义：**
```
吕布就是无敌的。字面意思。

  1. 他的必杀不可化解——牵制克必杀这个铁则，在他身上不成立
  2. 他化解必杀概率极高(75%)且效果极强(-70%)——即使失败也被威压削弱(-20%)
  3. 他不需要防御——格挡/闪避/牵制全是多余的，他只需要进攻
  4. 他的必杀三连击一套≈89伤害——任何武将吃满必死
  5. 即使奇迹发生——无双"单挑不败"保证他不会死

结论: 能"不输"给吕布的唯一方式，是撑到10回合平局。
  这正是天下无双的真正含义——不是不会输，而是**没人能赢**。
```

### 8.8 结局与战场影响

#### 8.8.1 结局判定

一方 HP ≤ 0 时进入结局判定：

```
被斩概率 = 15% + 勇猛差(攻方-守方)×2% + 历史仇人+15% + 专属(无双+10%)
被俘概率 = 40% - 体力剩余量×0.2（体力越多越不易被俘）
  优先判定被斩 → 被俘 → 逃脱
  例外: 历史宿命对决（关羽斩华雄等）必然斩杀

平局: 双方同时HP≤0，或10回合未分胜负
  双方均不斩/俘，士气各+3（惺惺相惜）
```

#### 8.8.2 战场影响

| 结局 | 对败方 | 对胜方 | 观众效应 |
|:----:|--------|--------|:--------:|
| **斩杀** | 武将死亡，部队溃散(兵力归0) | 士气+15，功绩+30 | 友军士气+5，敌军士气-10 |
| **俘虏** | 武将变为俘虏(所在部队溃散) | 士气+10，功绩+20 | 友军士气+3，敌军士气-5 |
| **逃脱** | 体力归1，不可再战，撤回本阵 | 士气+5，功绩+10 | 双方士气±2 |
| **平局** | 各回本阵，体力不变 | 士气+3，功绩+5 | 双方士气+3 |
| **投降** | 等同俘虏(概率出现在<15%体力时) | 士气+8，功绩+15 | 敌军士气-8 |

#### 8.8.3 功绩/体力消耗

```
胜方: merit + (斩:30 / 俘:20 / 逃:10 / 平:5)
败方: 无功绩
单挑体力消耗: 胜方 -15，败方 -30（叠加于单挑中HP损失所对应的体力换算）
```

### 8.11 单挑暴击/反击/连击（独立规则）

> 以下规则**仅作用于单挑**（§8），与战场部队系统（§6.2~6.4）完全隔离。

#### 8.11.1 暴击（单挑）

```
单挑暴击率 = 武力/50 + 武器修正(刀+5%, 钝器+10%) + 指令修正(猛攻+10%, 周旋+5%)
  + 技能修正(勇武Lv1~5: +2%~+10%)
  + 特性修正(豪勇Lv1~5: +5%~+30%)
  + 专属修正(武圣+30%, 无双+20%)
  + 宝物修正(方天画戟+10%, 青龙偃月刀+5%)
  + 运气/5

上限 70%，下限 3%（文官保底）
暴击伤害倍率 = 1.5
  武圣 → 2.5（专属覆盖基础）
  刚力(Lv1~5) → 最终倍率 ×(1+0.10~0.40)

暴击惩罚: 被克指令方向 -10%；牵制指令 -5%（牵制不求暴击，求稳）
必杀指令: 不暴击（固定MAX）
格挡/闪避: 不暴击（防御型）
```

#### 8.11.2 反击（单挑·反手一击）

```
触发条件: 指令为「格挡」且未被周旋克制（周旋克格挡 → 反手失效）
基础触发率: 30%
  + 技能修正(固守Lv1~5: +5%~+25%)
  + 特性修正(豪勇Lv1~5: +10%~+40%, 铁壁+3%~+15%)
  + 专属修正(恶来+30%, 刚烈→必反)

反手一击伤害 = 正常伤害 × 0.5
反手一击可暴击（独立roll，暴击率同单挑暴击率）

专属强化:
  张飞·刚烈 → 格挡后必反击 + 必暴击 + 伤害×1.0(取代×0.5)
  典韦·恶来 → 每回合可反击2次
  周旋克制格挡 → 反手一击完全失效（被周旋打乱防守节奏）
```

#### 8.11.3 连击（单挑）

```
触发条件: 仅通过专属技能触发（无通用连击率公式）:
  吕布·无双: 三连击（第二击×0.7, 第三击×0.5，可独立暴击）
  太史慈·天义: 二连击（第二击×0.8，可独立暴击）
  关羽·武圣: 击败对手后下回合(若单挑未结束)必连击
  赵云·龙胆: 每回合连击率+10%(累加至上限60%)

通用连击技能加成(仅对拥有连击专属的武将生效):
  急攻Lv1~5: +3%~+15% 连击率
  豪勇Lv1~5: +5%~+25% 连击率
```

#### 8.11.4 防循环规则

| 单挑事件 | 可暴击 | 可连击 | 可反击(反手) |
|:--------:|:------:|:------:|:------------:|
| 普攻(猛攻/牵制/周旋) | ✅ | —（由专属决定） | — |
| 反手一击 | ✅ | ❌ | ❌（已反过一次） |
| 连击(二/三) | ✅（独立roll） | ❌ | ❌ |
| 必杀 | ❌ | ❌ | ❌ |
| 暗袭 | ❌ | ❌ | ❌ |
| 格挡 | ❌ | ❌ | ✅（反手判定） |
| 闪避 | ❌ | ❌ | ❌ |

### 8.13 单挑决策（全自动·引擎内部）

> 沿用经典三国策略游戏模式：**引擎替双方所有武将自动决策**。玩家不参与指令选择，也不需要替己方武将选指令。
> 
> 玩家只做两个决定：
> 1. **是否主动发起单挑**（点击【单挑】按钮 → 选目标）
> 2. **是否接受对方的单挑**（确认框 → 接受/拒绝）
> 
> 进入单挑后，一切由引擎根据以下规则自动推演。

#### 8.13.1 发起/接受决策

```
AI 发起单挑判断:
  if 己方武力 - 敌方武力 ≥ 10 → 主动发起（概率 80%）
  if 己方武力 > 敌方武力 → 概率 50%
  if 己方武力 ≤ 敌方武力 → 概率 10%（除非专属加成）
  且己方气力 ≥ 20、体力 ≥ 40%

AI 接受/拒绝:
  if 敌方武力 - 己方武力 ≥ 15 → 拒绝（-15士气保命）
  if 己方体力 < 40% → 自动拒绝
  else → 接受
```

#### 8.13.2 指令选择（引擎内部·双方通用）

```
依据武将性格加权（发扬自经典版本）：

| 性格 | 猛攻 | 牵制 | 必杀 | 格挡 | 闪避 | 周旋 | 暗袭 |
|:----:|:----:|:----:|:----:|:----:|:----:|:----:|:----:|
| 勇敢 brave | 30% | 15% | 15% | 15% | 10% | 10% | 5% |
| 冷静 calm | 10% | 25% | 5% | 30% | 20% | 5% | 5% |
| 莽撞 reckless | 35% | 5% | 25% | 10% | 5% | 15% | 5% |
| 谨慎 cautious | 10% | 20% | 5% | 25% | 25% | 10% | 5% |
| 大胆 bold | 25% | 15% | 20% | 10% | 10% | 15% | 5% |
| 温和 gentle | 15% | 25% | 5% | 20% | 20% | 10% | 5% |

动态调整:
  若对方上回合使用必杀 → 牵制权重+30%（预判对方可能继续必杀）
  若对方上回合使用牵制 → 猛攻权重+25%（克牵制）
  若对方HP < 30% → 必杀权重×2（一击定胜负）
  若气力 ≥ 50 → 必杀权重×2（莽撞/勇敢额外×1.5）
  若己方HP < 30% → 格挡/闪避权重各+20%，猛攻-20%
  若暗袭未用且对方先手 → 暗袭权重+15%
  若对方连续2回合格挡 → 周旋权重+30%（克制格挡）
```

#### 8.14.1 宿命对决详表

> 发扬自经典三国策略游戏 的戏剧化叙事精神：当特定武将在历史对应场景中相遇时，触发
> **强化叙事 + 专属规则 + 结局修正**，使单挑更具传奇色彩。
>
> 优先级高于普通单挑规则（覆盖或扩展 §8.3~§8.13 的通用规则）。

##### 1. 三英战吕布 — 吕布vs关羽+张飞+刘备（联军侧）

| 项 | 设定 |
|:--:|------|
| **触发条件** | 虎牢关之战（剧本事件触发）；或吕布与刘备/关羽/张飞在战场上相邻，且吕布武力最高 |
| **特殊规则** | **三将接力战**：刘关张依次上场，当前一人 HP≤30% 时下一人自动替入（不消耗额外行动）；替入时吕布不恢复 |
| **HP池** | 刘关张共享 **350 HP**（刘备 100 + 关羽 120 + 张飞 130），吕布 **150 HP** |
| **连击禁止** | 吕布的三连击在本战中禁用（演义中未连杀三人）→ 改为**二连击**（第二击×0.7） |
| **士气联动** | 刘关张每替入一人，吕布士气 -5（兵力分散焦虑）；关羽上场时额外 -5（武圣威压） |
| **叙事模版** | 替入时：{新将}「休伤我兄弟！」拍马舞刀杀入战团 |
| **结局修正** | 若吕布战败 → 必定逃脱（赤兔马加成），不会被斩/俘；若吕布战胜 → 刘关张无人阵亡（亦逃脱） |

**设计意图**：还原演义中"三将轮流战吕布，吕布终不敌"的名场面，同时给吕布玩家翻盘机会——若能在前两将消耗中节省体力，第三将有反杀可能。

**设计意图**：还原演义中最著名的秒杀名场面。这是一个"首回合定胜负"的特殊规则——成功则速杀，失败则关羽陷入苦战。

### 8.16 设计原则与集成备忘

```
1. 独立性: 单挑所有数值与 §6 战场系统完全隔离
```

### 8.17 单挑大会（S19 独立系统）

> 独立于战场单挑（§8.1~8.16）的周期性锦标赛系统，对标经典三国策略游戏闘技大会并深度扩展。
> 引擎使用与战场单挑完全相同的全自动结算规则（§8.5~8.11），但独立于战场状态——大会中的胜负不影响战场上的生死。

#### 8.17.1 定位

| 项目 | 说明 |
|:----:|------|
| **系统归属** | S19 单挑大会（独立系统，12-system-map 新增） |
| **触发方式** | 每年 1 月由系统自动触发（非 S14 事件） |
| **结算引擎** | 复用 §8.5~8.11 全自动单挑规则（三向克制·7指令·隐藏属性·技能·武器） |
| **独立性** | 大会胜负**不导致武将死亡**（唯伤不杀）；不影响战场单挑冷却 |
| **与战场单挑的区别** | 大会是"竞技"非"死斗"——不斩杀、不俘虏、不影响战场兵力 |

#### 8.17.2 大会模式

玩家在每年大会触发前选择模式，选定后不可更改：

| 模式 | 吕布规则 | 适用场景 |
|:----:|----------|:--------:|
| **无限制** | 吕布以天下无双型全状态参赛——必杀不可化解、75%被动化解率、三连击全开 | 追求演义真实感，让其他人争第二 |
| **公平竞技** | 吕布禁用天下无双被动（天下无双型降级为猛将型）：必杀**可被化解**、**无被动化解**、三连击保留（但不可暴击） | 想看到底谁是真功夫，给其他人夺冠机会 |

> 建议首次游玩默认"公平竞技"——等玩家熟悉系统后，再切"无限制"体验吕布的压迫感。

#### 8.17.3 举办与参赛

**举办时机**：
- 每年 **1 月** 触发公告
- 若当年有势力灭亡/统一等重大事件 → 跳过当年大会
- 举办城市：上届冠军所在势力的都城（首届在洛阳）

**参赛资格**（同时满足）：
```
武力 ≥ 70
体力 ≥ 80%（若赛前体力不足，自动跳过）
非重伤/非任务中/非在押
未参加当年大会的其他武将（每人每年限1次）
```

**各势力名额**：
| 势力规模 | 城数 | 名额 |
|:--------:|:----:|:----:|
| 大 | ≥ 8 城 | 5 人 |
| 中 | 4~7 城 | 3 人 |
| 小 | ≤ 3 城 | 2 人 |
| 在野 | — | 1 人（如有在野武将达到条件） |

> 玩家手动选择派谁参加（从符合条件的武将列表中勾选）。
> 未指派时系统自动按武力从高到低补满名额。
> 武将可**拒绝**参加（忠诚<80 或 相性与君主不合）→ 报名-15 忠诚。

#### 8.17.4 赛制

**规模**：16 人单败淘汰（可扩展至 32 人）

**种子制**：
```
第1种子: 上届冠军（首届按武力最高）
第2种子: 本届武力最高（不重复）
第3~4种子: 武力第2~3高
→ 种子分散四区，首轮不碰面
```

**对阵表**（标准 16 人淘汰赛树形）：
```
1区: ①种子 vs 16 → ⑨ vs ⑧ → 准决赛
2区: ⑤种子 vs 12 → 13 vs ④ → 准决赛
3区: ③种子 vs 14 → 11 vs ⑥ → 准决赛
4区: ⑦种子 vs 10 → 15 vs ② → 准决赛
```

**体力规则**：
- 每轮之间**体力不恢复**（连战消耗，给黑马爆冷机会）
- 每轮结束后可使用道具恢复（持有"体力丸"类宝物时）
- 体力 < 30% 时全属性 -20%（同 §8.6.3）
- 体力 < 15% 时可投降（不扣名声）

**平局加赛**：
- 10 回合未分胜负 → 加赛 3 回合
- 加赛仍平 → 先手方换位再赛
- 第三次平局 → 双方均淘汰（空出晋级位）

#### 8.17.6 押注系统

**机制**：
- 每轮对阵公布后 → 弹窗押注窗口
- 押注消耗势力**金**，限额 = 势力当前金 × 20%
- 赔率动态计算：

```
baseOdds = 防守方武力 / 攻击方武力
（例：武100吕布 vs 武75 → 100/75 = 1.33 倍）
爆冷修正: 若武力差 ≥ 15 且低武力方获胜 → 赌注 ×3
历史修正: 若两人过去交过手（大会/单挑）→ 结合战绩微调
```

**交互**：
```
[孙策(92) vs 张辽(85)]  赔率: 1.08
 押孙策: [100金] [500金] [1000金]
 押张辽: [100金] [500金] [1000金]  (爆冷赔率×3)
 [跳过本轮]
```

### 11.1 士兵俘虏

```

降兵数 = 敌方战前总兵力 × 降兵率

处置：
  [收编] → 编入己方(兵质-1级,士气30~50,逃兵5%/季×4季)
  [遣散] → 释放回乡, 所在城市人口+降兵×0.2
  [坑杀] → 声望-50, 周边城民心-5, 敌对我仇恨
  [劳役] → 加速城墙建设30%/季
```

```
粮草 → 敌剩余粮草×0.5~0.8
宝物 → 敌主将装备30%概率缴获

```

```
战后管制:
  [镇压] 驻军≥人口5%→民心恢复-5, 叛乱-90%
```

## 十二、战役地图节点

> 战役层（Campaign Layer）是游戏主玩法所在：玩家在大地图上组建部队、行军、建造设施、攻城略地。
> 战斗由科学算法自动结算（§十七），不再需要进入六角网格手动操作。
> 0-A 阶段以 30 治所验证流程，后续扩容至县城级节点。

### 12.1 节点类型

| 类型 | 0-A 状态 | 说明 |
|:----:|:--------:|------|
| **大城**（治所） | ✅ 30 城 | 郡国首府，有城墙/驻军/内政功能 |
| **县城** | 🔜 0-B | 郡国下辖县（如河南尹21县），无城墙，产粮/征兵 |
| **关隘** | 🔜 0-B | 封锁道路，易守难攻（阳平关/虎牢关/函谷关等） |
| **港口** | 🔜 0-B | 水域转运点，水军通行条件 |
| **设施** | 玩家建造 | 营寨/粮仓/瞭望塔等，见 §十五 |

### 12.3 道路网络

```
节点通过邻接表连接。行军沿道路逐节点进行，每个节点消耗 1 回合。

  例：宛(19) ── 襄阳(17) ── 江陵(18)
      从宛出征江陵：宛→襄阳(第1回合)→江陵(第2回合到达)

关隘封锁规则：
  关隘节点驻守敌军时，封锁所有邻接道路
  攻方必须先攻破关隘才能继续前进
  可绕道（如果存在其他道路连接）

0-A 实现：
  复用现有 city-roads.ts 30 城邻接表
  官道=节点邻接关系
  后续扩容时基于实际地理关系生成
```

### 12.4 郡国体系

```
0-A：30 城 = 30 郡国各 1 治所

0-B 扩容后：
```

> 在战役地图上以行军箭头标记移动，包含兵力/补给/士气/设施等状态（§5.5 编成规则在此落地）。

### 13.2 编成流程（战役视角）

```
出征编成步骤：
  1. 选主将 → 决定兵种/阵型/基础战力
  2. 选副将 → 依武官官职决定数量（§5.5.1）
  3. 选参谋（智≥85）+ 副参谋（大将军/君主可设）
  4. 分配兵力（主将占比≥50%，副将均分剩余）
  5. 分配 Squad 站位（可自动/手动）
  6. 带粮（取该城库存，上限由部队规模决定）
  7. 择目标 → Army 出现在出发城节点 → 进入行军阶段
```

### 13.3 Squad 五部阵位

```
每支 Army 的 Squad 分配至五个阵位之一：

  阵位    作用                            适宜角色
  ──────  ───────────────────────────────  ─────────────
  先锋    开道·拦截·追击·先锋战             猛将/骑兵将领
  中军    主将所在·主力决战                  主将
  左翼    侧击·包抄·防包抄                  均衡型副将
  右翼    侧击·包抄·防包抄                  均衡型副将
  后卫    押运粮草·殿后·防袭扰              谨慎型/防御型

阵位影响：
  - 先锋 vs 先锋 触发"先锋对决"（士气影响+单挑概率↑）
  - 侧击（左/右翼位 对 敌侧翼）→ 自动战斗加 +10% 伤害
  - 后卫被袭 → 粮草损失 + 士气降
  - 阵位可在行军途中调整（消耗 1 回合，受阵型限制）

参谋/副参谋不占阵位，以幕僚身份附于中军。
```

### 13.4 补给系统

```
每回合消耗：
  foodPerTurn = floor(troops / 100) × 3 × terrainMul

  terrainMul: 平原/友好城 1.0 | 山地/敌境 1.5 | 严寒/酷暑 1.3

缺粮效果：
  food ≤ 0 当回合 → 士气 -15，组织度 -10
  food ≤ 0 持续 → 每回合士气 -5，组织度 -5
  food ≤ 0 持续 3 回合以上 → 溃散概率 +20%/回合

补给来源：
  - 己方城市（行军经过时自动补满）
  - 攻占城市（获取该城库存 50%）
  - 粮仓设施（§15.6，延长补给线）
  - 军屯田（§5.8.1，驻守时产粮）

补给线概念（S17 釜底抽薪联动）：
  每支 Army 从出发城/最近己方城拉补给线
  补给线被切断（敌军占领中间节点）→ 缺粮状态
  参谋技能「后勤」可减轻补给线中断影响
```

### 13.5 部队品质在战役层

```
士气·组织度·经验·疲劳 四维在战役层的表现：

士气 (0-100)：
  影响自动战斗公式中的士气系数
  行军 -2/回合 | 缺粮 -15/回合 | 胜仗 +10 | 败仗 -20
  参谋「激励」指令 → +15（消耗参谋体力 15）
  主将特性「威压」→ 敌士气衰减 +5%/回合

组织度 (0-100)：
  影响阵型加成有效率和溃散概率
  行军 -1/回合 | 战斗后 -5~15 | 驻守 +5/回合
  低于 30 → 溃散概率 +30%

经验 (0~3000, Lv1~7)：
  影响战力公式中的经验系数（Lv1=0.8, Lv7=1.3）
  战后存活 +20xp | 胜仗 ×1.5 | 全灭归零

疲劳 (0-100, 越高越疲劳)：
  每日行军 +10 | 战斗 +20 | 扎营 -15/回合 | 驻守 -10/回合
  ≥80 → 士气衰减 ×2，战力系数 ×0.85
  参谋「撤退休整」指令 → 疲劳 -30（消耗参谋体力 10）
```

### 13.6 参谋在战役层的角色

| 能力 | 消耗 | 效果 | 条件 |
|:----|:----:|------|:----:|
| **激励** | 参谋体力 -15 | 全队士气 +15，不可叠加 | 体力 ≥ 30 |
| **陷阱** | 参谋体力 -20 | 当前节点布陷阱，敌通过受智×1.5伤 | 智 ≥ 90，体力 ≥ 40 |
| **撤退休整** | 参谋体力 -10 | 本回合有序撤退，疲劳 -30，不触发溃散 | 体力 ≥ 20 |
| **斥候扩大** | 无消耗 | 部队视野半径 +1 节点 | 常驻 |
| **献策** | 无消耗 | 每回合概率触发，影响 AI 决策权重 | 常驻 |

## 十四、总军师系统

> 总军师（Grand Strategist）是势力层面的战略决策者，与每军的参谋（Advisor）层级不同。
> 总军师只有一位，决定势力的整体战略方向。

### 14.1 层级定位

```
势力层 ───── 总军师 ────────────────────────
  战略级：决定打谁/守哪/主攻方向
  影响：AI 决策权重 · 资源分配 · 外交倾向 · 战役态势
  人数：势力仅 1 位
  典型：诸葛亮(刘备)·荀彧(曹操)·周瑜(孙权)·沮授(袁绍)

战役层 ───── 参谋（每军一位）─────────────────
  战役级：陷阱·斥候·激励·撤退休整
  影响：单个 Army 作战效率
  人数：每 Army 1 参 + 1 副参

战术层 ───── 战场计策（火计等）───────────────
  战术级：战斗中即时施放
  影响：单次战斗局部效果
```

### 14.2 任命条件

```
条件：
  智 ≥ 90（初始）/ 智 ≥ 85（通过功绩解锁）
  与君主相性差 ≤ 50
  不可兼任军中参谋（一人不能同时在总军师和军参谋位）
  每势力仅限 1 人，解职后才可另任

解职原因：
  君主主动解职
  总军师被俘/死亡
  总军师忠诚 ≤ 60（概率自动辞职）
```

### 14.3 态势决策

总军师设定势力的**战略姿态**，每季可切换一次（冷却 1 季）：

| 态势 | 效果 | 适用场景 |
|:----:|------|---------|
| **进攻** | Army 士气 +5，出征粮耗 -20%，AI 主动出征概率 ×1.5 | 兵力优势/抓住战机 |
| **防守** | 城池驻军效率 +20%，设施建造速度 +25%，AI 出征概率 ×0.5 | 内政发展期/多线受压 |
| **发展** | 内政指令效果 +15%，征兵金耗 -20%，AI 外交友好倾向 +10 | 休养生息/新占城池巩固 |
| **隐忍** | 被宣战概率 -30%，外交赠礼效果 ×1.5，计谋成功率 +10% | 势力弱势/韬光养晦 |

**总军师智力影响态势效果**：
```
实际加成 = 基础加成 × (1 + (总军师智 - 85) / 50)
  智 90 → 105% | 智 100 → 130% | 智 70（不合格）→ 70%
```

### 14.4 战略献策

总军师每回合概率触发献策事件，提供战略建议：

```
触发概率 = 15% + (总军师智 - 80) × 0.5%
  智 90 → 20% | 智 100 → 25%

献策类型（随机选一）：
  - 出征建议："臣料敌将XX可击，宜出XX城攻XX"
  - 内政建议："今宜修XX城之农，以备来年军资"
  - 外交建议："可遣使结好XX，共敌XX"
  - 识破预警："臣观敌势，恐有XX之计"
    （抵消敌方总军师一次战略计谋）
  - 人事建议："XX乃大才，现流落XX，宜速招之"
```

AI 势力自动采纳献策（按概率），玩家可选采纳/忽略。采纳后获得对应效果加成。

### 14.5 总军师对决

双方总军师在战役层存在隐性对抗：

```
每回合概率触发：
  己方总军师智 ≥ 对方总军师智 + 15 → 识破对方一次战役计谋（S17 L2）
  己方总军师智 ≥ 对方总军师智 + 30 → 反制：对方计谋无效，己方获知对方动向

态势对抗：
  己方进攻 vs 对方防守 → 主攻方加成 = max(0, (己军师智 - 敌军师智) / 100)
  该加成直接作用于自动战斗算法的「策略修正」项
```

```
② 防御设施（营寨/栅栏/壕沟）
   - 行军停驻时可选"扎营" → 自动建造营寨（1回合）

③ 机关设施（陷阱/瞭望塔——参谋专属）

④ 补给设施（粮仓）
   - 占领节点后 1 回合建造
   - 粮仓只对建造方有效，敌方占领后摧毁
```

### 15.3 设施耐久与摧毁

```
设施受攻击判定：
  被敌方 Army 经过 → 概率受损（受经过部队规模影响）
  被敌方 Army 攻击（攻城阶段）→ 纳入自动战斗结算
  设施无独立驻军，不可移动

耐久公式：
  baseDurability = 500 × (1 + 建造者统率/200)
  技能加成：筑城Lv → +20%/级
  特性加成：铁壁 → +40%

摧毁规则：
  营寨/栅栏/壕沟 → 耐久归零后消失
  攻城器械 → 部分耐久仍可用（威力 × 剩余耐久%）
  陷阱 → 触发一次后消失（不论是否命中）
  粮仓/瞭望塔 → 被敌方占领后直接摧毁
```

### 16.2 驻守阶段

```
驻守在城市或设施中，不移动。

可操作：
  ✓ 编成调整（换将/改阵型/分兵）
  ✓ 补充兵力（从该城征兵）
  ✓ 补充粮草（从该城调粮）
  ✓ 建造设施（§十五）
  ✓ 军屯田开关（§5.8.1）
  ✓ 出征/调遣
  
不可操作：
  ✗ 攻击
  ✗ 围城
```

### 16.3 行军阶段

```
沿道路节点前进，每回合移动一个节点。

可操作：
  ✓ 改变目标（重新规划路径）
  ✓ 扎营（到达节点后建造营寨）
  ✓ 派遣斥候（参谋能力，提前查看前方节点情报）
  ✓ 撤退（原路返回）
  ✓ 调整 Squad 阵位

自动事件：
  目标节点有敌方 Army → 进入野战接战
  目标节点有敌方关隘 → 到达后进入围城
  敌 Army 在中途节点拦截 → 进入野战接战
  经过己方城 → 自动补粮（取该城库存 50%）

补给消耗：
  每回合粮耗 = floor(troops/100) × 3 × 地形系数
```

### 16.4 野战阶段

```
两支敌对 Army 在同一节点相遇时触发。

流程：
  1. 先判定先锋对决（双方先锋将智力/武力较量）
  2. 进入自动战斗结算（§十七）
  3. 结算后根据结果进入战后阶段

特殊规则：
  - 守方可选"避战撤退"（组织度 -20，保留兵力进入相邻节点）
  - 攻方可选"追击"（兵力优势 > 2:1 时触发，士气 +5）
  - 单挑概率在先锋对决阶段判定
```

### 16.5 围城阶段

```
Army 到达敌方城市/关隘节点时进入围城。

围城每回合可选操作：

  [围困] — 默认持续
    效果：守方士气 -5/回合，民心 -2/回合
    我军：建造攻城器械中（如果正在建造）
    
  [建造器械] — 选择副将开始建造
    选项：冲车(2回合) / 云梯(2回合) / 井阑(3回合) / 投石车(3回合)
    多个器械可依次建造，不可并行
    器械建造完成后自动装备到 Army
    
  [劝降] — 概率性
    基础成功率 = 10% + (主将魅 - 守将魅) × 0.5% + 围城回合 × 2%
    上限 60%
    总军师「隐忍」态势下 +10%
    势力特点影响（曹操"挟天子"→ 劝降 +15%）
    
  [强攻] — 进入攻城自动战斗
    进入§十七自动战斗，加入攻城器械修正
    强攻失败 → 我军伤亡 ×1.5，士气 -20
    强攻成功 → 破城，进入战后阶段
    
  [撤围] — 解除围城
    返回行军阶段，士气 -10
```

**围城特殊规则**：
```
断粮判定：
  围城 3 回合后 → 守军粮耗完 → 守军士气额外 -10/回合
  被围城市无相邻己方城连接 → 判定为完全包围 → 守军不可撤退

守方突围：
  守方可选"突围"（兵力 ≤ 40% 时自动倾向突围）
  突围战斗按§十七结算，侧重视野修正（守方 -20%）
```

### 16.6 强攻阶段

```
强攻 = 攻城自动战斗

参数修正：
  攻方战力 += Σ(攻城器械攻击修正)
  冲车: +30% | 云梯: 忽略城墙 | 井阑: 守方防 -20% | 投石车: +50%

特殊事件概率：
  单挑（攻城时守方将领出战概率 ↑）
  城门被破（有冲车时）
  城墙塌陷（有投石车时）
  守方投降（士气 ≤ 20 时概率判定）
```

### 16.7 战后阶段

```
战斗/围城结束后处理：

胜利方：
  占领城市（城池归属变更）
  缴获：该城金粮 50%，宝物概率缴获
  驻军：剩余兵力可驻防（部分兵力留下，Army 返回出发城）
  部队经验 +50xp，士气 +10

失败方：
  撤退：残兵按比例返回出发城
  被全歼：该 Army 解散，主将/副将概率被俘/阵亡
  败将判定：
    主将逃脱（概率 = 智力/200 + 武力/200）
    副将跟随主将（或各自判定）
    参谋逃脱（概率 = 智力/150，≥80 必逃）

僵持：
  双方均损失惨重 → 各自撤回原节点
  下回合可选择再战

郡国归属（0-B 县城扩容后）：
  占领治所后，县城归属由算法决定（§17.6）
```

## 十七、自动战斗算法

> 战役层所有战斗通过统一算法自动结算，不进入六角网格。
> 算法以确定性公式为主，附加受控随机因子，确保结果可预期、可解释。

### 17.1 设计原则

```
1. 确定性优先：相同输入 → 相同输出（除明确标注随机的项）
2. 因子可追溯：战斗报告展示每项修正的具体来源
3. 多回合模拟：非一锤定胜负，模拟多轮攻防交互
4. 单挑作为事件：在推演中概率触发，影响士气/将被斩判定
5. 郡国归属独立算法：战斗胜利不一定全占郡国
```

### 17.2 战力公式

```
总战力 = 基础战力 × 编成修正 × 状态修正 × 环境修正 × 计谋修正

一、基础战力
  basePower = troops × unitPower(unitType)
  
  unitPower 表：
    轻步 1.0 | 重步 1.3 | 长枪 1.2 | 弓 0.9 | 弩 1.1
    轻骑 1.2 | 重骑 1.6 | 骑射 1.0
    走舸 0.8 | 蒙冲 1.2 | 楼船 1.5

二、编成修正
  commandMod = mainMod + subMod + advisorMod + formationMod
  
  mainMod = 1 + (主将统率 + 主将武力/2) / 200
  subMod = Σ(副将统率 × 0.3 × relationCoeff) / 副将数
  advisorMod = 参谋智力 / 200（仅战场计策加成）
  formationMod = 阵型攻防修正 / 100（从§4.1取值）

三、状态修正
  statusMod = moraleCoeff × orgCoeff × expCoeff × fatigueCoeff
  
  moraleCoeff = 0.4 + 士气/100 × 0.6        (士气0→0.4, 100→1.0)
  orgCoeff = 0.5 + 组织度/100 × 0.7          (组织度0→0.5, 100→1.2)
  expCoeff = 0.8 + (经验/400)                 (Lv1→0.8, Lv7→1.3)
  fatigueCoeff = 1.0 - 疲劳/200               (疲劳0→1.0, 100→0.5)

四、环境修正
  envMod = terrainMod × weatherMod
  
  terrainMod（以攻击方为准）：
    平原 1.0 | 森林 0.9 | 山地 0.8 | 水域 0.7(非水军)/1.2(水军)
  
  weatherMod：
    晴 1.0 | 雨 0.9 | 雾 0.85 | 雪 0.9

五、计谋修正
  stratagemMod = 1.0 + Σ(S17 计谋效果)
  
  釜底抽薪（断粮）：0.7（该军）
  调虎离山（诱敌）：0.85（被诱方）
  暗渡陈仓（偷袭）：1.2（奇袭方）
  借刀杀人：0.9（目标方）
  趁火打劫：1.15（趁乱方）
  （效果值从S17设计取值，此处为示例）

六、攻城修正
  攻城战额外因子：
  siegeMod = 1.0 + siegeEquipBonus - wallPenalty
  
  siegeEquipBonus（取已建造的最高级）：
    冲车 +0.3 | 云梯 +0.2(忽略城墙) | 井阑 +0.25(降敌防) | 投石车 +0.5
  
  wallPenalty（城墙）：
    大城 -0.3 | 关隘 -0.4 | 无城墙 0

七、总战力
  finalPower = basePower × commandMod × statusMod × envMod × stratagemMod × siegeMod
```

### 17.3 战斗推演

```
战斗不是一次计算定胜负，而是模拟多回合攻防：

模拟回合数 R = 3 + floor(双方总战力差 / 1000)，上限 10 回合

每回合流程：
  1. 攻方战力 × 回合随机浮动(0.9~1.1)
  2. 守方战力 × 回合随机浮动(0.9~1.1)
  3. 战力比 = 攻方 / 守方
  4. 伤亡比例 = 0.05 + (战力比 - 1) × 0.1（攻方胜利时）
     或 = 0.05 + (1/战力比 - 1) × 0.1（守方胜利时）
  5. 双方兵力递减：troops = troops × (1 - 伤亡比例)
  6. 单挑事件判定（见§17.4）
  7. 士气变化：胜方 +5，败方 -10
  8. 组织度变化：双方 -5

提前结束条件：
  一方兵力 ≤ 初始 30% → 溃散判定
  一方主将被斩 → 该方立即战败
  单挑导致主将被阵斩 → 立即战败

最终结果判定：
  winner = 最后一回合战力比 > 1.0 ? 攻方 : 守方
  将领伤亡/被俘按兵力损失比例概率判定
```

### 17.4 单挑在算法中的角色

```
单挑作为自动推演中的事件触发，不打断流程（仅记录日志）：

触发概率：
  baseChance = 5% + (双方主将武力差绝对值) / 10
  先锋对阵 +10%（先锋阵位）
  守城方低士气 +5%（士气 ≤ 40）
  吕布参战 +15%（"虓虎之勇"）

单挑效果（影响战力评估）：
  己方胜 → 士气 +15，敌方士气 -10
  己方败 → 士气 -10，敌方士气 +5
  主将被斩 → 立即战败（不计剩余回合）
  主将被俘 → 立即战败 + 该将被俘

单挑演出：
  算法计算出单挑后，前端可展示叙事文本
  玩家可选择"观看"（弹出单挑结算叙事）或"跳过"
  观看不改变结果，仅为演出
```

### 17.6 郡国归属算法

> 攻占治所后，该郡国下属县城是否全部/部分投降，由算法动态决定。

```
投降率 = 基础 20%
  + 双方兵力差 / 总兵力 × 30（最多 +30%）
  + (攻方士气 - 守方士气) × 0.2（最多 ±10%）
  + 主将魅差 × 0.5（最多 ±10%）
  + 攻方总军师智 / 200（最多 +10%）
  + 守方总军师智 / -200（最多 -10%）
  + 势力特点修正（曹操"挟天子"+15%，刘备"仁德"+10%等）
  + 围城回合 × 3%（最多 +15%）
  - 守方特性"不屈"-20%，"刚烈"-15%

判定结果：
  ≥ 80% → 全郡归顺，所有县城自动归属
  ≥ 50% → 半数县城归顺，其余保持中立/继续抵抗
  ≥ 20% → 仅有少数偏远县归顺，多数继续抵抗
  < 20% → 全郡抵抗，无县城归顺（需逐个攻占）

抵抗状态：
  继续抵抗的县城 → 兵力归零，但不投降
  每季度概率再次判定投降率（受民心/围城/时间影响）
  可逐个攻占
```
